## 摘要
近期可泛化的3D高斯泼溅模型推动了长序列新视角合成的发展，但代价是大量冗余计算。我们发现可以基于两点观察缓解冗余：(i) 高质量新视角合成并不严格要求高精度几何；(ii) 外观学习通常比几何恢复更容易。受此启发，我们提出一种非对称架构，将几何与外观建模解耦。几何分支使用大部分参数处理粗粒度令牌进行多视图重建，而外观分支则用显著更少的参数处理细粒度令牌以捕获细节。两个分支通过双边连接交互，为各自任务提供相互指导。这种任务感知的非对称性减少了计算冗余，更合理地分配计算，从而提高了参数效率，使较小模型也能实现强劲性能。在32视图960P输入上，我们的模型能与基于优化的方法相媲美，同时速度提升近800倍，并以明显更少的参数和更低的训练/推理开销超越了当前最先进可泛化模型的零样本性能，实现了整体效率的提升。

## Abstract
Recent generalizable 3D Gaussian Splatting models have advanced long-sequence novel view synthesis (NVS), but at the cost of substantial redundant computation. We identify that the redundancy can be mitigated based on two observations: (i) high-precision geometry is not strictly required for high-quality NVS; (ii) appearance learning is generally easier than geometry recovery. Motivated by these insights, we propose an asymmetric architecture that decouples geometry and appearance modeling. The geometry branch processes coarse-grained tokens with most of the parameters for multi-view reconstruction, while the appearance branch operates on fine-grained tokens to capture details using significantly fewer parameters. The two branches interact through bilateral connections, enabling mutual guidance for their respective tasks. This task-aware asymmetry reduces the computational redundancy and allocates the computation more judiciously, thereby increasing parameter efficiency and enabling smaller models to achieve strong performance. On 32-view 960P inputs, our model matches optimization-based methods while delivering nearly 800x speedup, and surpasses the zero-shot performance of state-of-the-art generalizable models with markedly fewer parameters and reduced training/inference overhead, achieving an overall efficiency improvement.

## 1. 论文的核心问题与整体含义

**研究动机与背景**  
- 通用（可泛化）3D Gaussian Splatting (3DGS) 模型在长序列、高分辨率新视图合成（NVS）中取得进展，但存在大量计算冗余，导致参数量和训练/推理开销巨大。  
- 先前的通用方法（如 LongLRM）将所有来自高分辨率图像的细小 patch token 都送入层叠的网络进行跨帧交互，计算代价随视图数和 token 数急剧增长。  
- 作者从两个关键观察出发：  
  (i) 高精度几何并非高质量渲染的必须条件——3DGS 通过 alpha 混合可弥补几何误差；  
  (ii) 外观属性的预测比多视图几何恢复更容易，只需将像素对齐的 RGB 证据映射到高斯中，无需密集的跨视图匹配。  

**整体含义**  
- 提出非对称架构 AsySplat，将几何建模与外观建模解耦，为不同任务分配不同粒度的 token 和参数量，从而大幅度削减冗余计算，提升参数效率，使小模型也能达到强劲性能。

## 2. 方法论：核心思想与关键技术细节

**核心思想**  
- 基于上述两个观察，将计算资源按任务难度和精度需求重新分配：  
  - 几何分支接收**粗粒度** token（大 patch size），但分配多数参数，专注于多视图几何重建；  
  - 外观分支接收**细粒度** token（小 patch size），但参数极少，仅进行帧内特征交互以捕捉细节。  
- 两分支通过**双边连接**相互传递信息，最终融合输出高斯场。

**关键技术细节**  
- **Token化**：  
  - 几何分支 patch size \(p_c = 16\)，产生粗粒度 token；  
  - 外观分支 patch size \(p_f = 8\)，产生细粒度 token。  

- **几何分支**（Geometry Branch）  
  - 分三个层级阶段（stage 1→2→3），token 维度逐步增大（192→512→896）。  
  - 每个阶段采用混合架构：双向 Mamba 层（缓解长序列交互） + 稀疏注意力层（Sparse Attention）。  
  - **稀疏注意力模块**（见图3）：  
    - 将 token 序列重整形为 2D token map，用卷积聚合局部信息；  
    - 应用 \(p_g \times p_g\) 均匀网格划分，将 token 重排为长度大幅缩短的序列；  
    - 在该短序列上执行自注意力，大幅降低复杂度和训练时间，同时保持上下文完整性。  
  - 注意：几何分支虽然采用粗粒度 token，但仍需大量参数以完成困难的跨视图匹配和多视图立体。

- **外观分支**（Appearance Branch）  
  - 同样分三个阶段，但 token 维度固定 \(d_f=256\)，且所有交互**仅限帧内**（intra-frame），不进行跨视图计算。  
  - 使用 MambaVision（高效视觉 Mamba 变体），每帧独立处理细粒度 token。  
  - 参数量约为总参数的 10 %。

- **双边连接模块**（Bilateral Connection）  
  - 在每个阶段的末尾，两分支进行信息交换：  
    - 粗粒度 token 通过交叉注意力吸收细粒度的高频细节；  
    - 细粒度 token 从粗粒度 token 获得几何上下文，辅助外观属性预测（如高斯尺度常与深度相关）。  
  - 所有交叉注意力均为帧内操作。

- **合并与高斯预测**  
  - 将两分支的 token map 对齐后相加，用步长卷积融合；  
  - 通过线性解码器预测每个像素对应的多个高斯属性（中心、不透明度、尺度、四元数、球谐系数）。  
  - 由于 patch size 不同，最终高斯数量显著减少（如 540×960 时从 8M 降至 4M）。

**算法流程简述**  
1. 输入 V 张图像 + Plücker 射线，分别 token 化为粗粒度和细粒度序列。  
2. 粗粒度 token 送入几何分支，经三阶段 Mamba+稀疏注意力处理；细粒度 token 送入外观分支，经三阶段帧内 MambaVision 处理。  
3. 每阶段末执行双边连接。  
4. 将两分支输出融合，解码为高斯场。  
5. 采用图像损失（MSE+感知损失）和深度监督（单目深度平滑 L1）端到端训练。

## 6. 主要结论与发现

- 非对称解耦设计能有效消除计算冗余，使参数效率显著提升。  
- 几何分支依赖粗粒度 token 并分配多数参数，足以支持高质量渲染；外观分支只需少量参数和帧内处理即能获得高保真外观。

- 参数量减少 30%，训练时间减少 30%，推理 FLOPs 减少 20%，高斯数量减少 50%，后优化速度提升 60%。  
- 在稀疏视图设定（2 视图 RealEstate10K）上同样具有竞争力，表现出良好的通用性。
