## 摘要
文本条件化的3D生成在图像和孤立物体方面进展迅速，但生成手物网格仍然具有挑战性：输出必须保持语言语义、跨视图一致性、物体几何、关节手形状和物理上合理的接触。我们提出了TextHOI-3D，一个分阶段框架，使用生成的多视图观察作为文本条件视觉生成与几何感知手物恢复之间的显式接口。TextHOI-3D为固定相机手物观察学习一个紧凑的VQ标记空间，使用CLIP条件化的视觉自回归模型从文本预测多视图视觉标记，并通过先验初始化、多视图联合优化和防穿透细化恢复统一的手物网格。该设计将语义生成与几何恢复分离，同时通过离散多视图表示保持两个阶段的连接。在HO3D衍生的评估中，与单视图设置相比，多视图设置将物体CD从17.26毫米降低到4.92毫米，穿透体积从5.3721立方厘米降低到0.2193立方厘米，同时改善了手部误差和表面F分数。这些结果支持多视图视觉标记作为文本驱动3D手物网格创建的有效中间表示。

## Abstract
Text-conditioned 3D generation has progressed rapidly for images and isolated objects, but producing a hand-object mesh remains challenging: the output must preserve language semantics, cross-view consistency, object geometry, articulated hand shape, and physically plausible contact. We present TextHOI-3D, a staged framework that uses generated multi-view observations as an explicit interface between text-conditioned visual generation and geometry-aware hand-object recovery. TextHOI-3D learns a compact VQ token space for fixed-camera hand-object observations, predicts multi-view visual tokens from text with a CLIP-conditioned visual autoregressive model, and recovers a unified hand-object mesh through prior initialization, multi-view joint optimization, and anti-penetration refinement. The design separates semantic generation from geometric recovery while keeping both stages connected by a discrete multi-view representation. On HO3D-derived evaluations, the multi-view setting reduces object CD from 17.26 mm to 4.92 mm and penetration volume from 5.3721 cm^3 to 0.2193 cm^3 compared with a single-view counterpart, while improving hand errors and surface F-scores. These results support multi-view visual tokens as an effective intermediate representation for text-driven 3D hand-object mesh creation.

# TextHOI-3D：文本驱动的三维手-物交互生成

## 1. 研究动机与核心问题
- **核心问题**：从自然语言描述直接生成手与物交互的三维网格（mesh）极具挑战，因为输出必须同时满足多重约束：语言语义保真、多视角一致性、物体几何稳定性、手部关节解剖合理性以及物理接触合理性（避免浮空和穿透）。
- **现有局限**：直接“文本→网格”的端到端方法难以处理手物接触区域的遮挡，且文本与三维手物对的配对数据稀缺。
- **整体思路**：将生成过程分阶段解耦，以“多视角图像”作为语义生成与几何恢复之间的显式中介。首先生成多视角手物图像，再从中恢复统一的三维手物网格。该方法旨在降低生成难度，同时保留图像中的轮廓、遮挡和接触线索以便驱动三维恢复。

## 2. 方法论
- **总体框架**：TextHOI-3D 由三个技术阶段构成：
  1. 离散多视角视觉表示学习（VQ-VAE）
  2. 文本条件多视角生成（CLIP 条件化的视觉自回归模型）
  3. 手物网格恢复（分割、补全、先验初始化、多视角联合优化、防穿透细化）
- **阶段一：离散多视角表示**
  - 将同一场景的多个固定相机视角的 RGB 图像沿通道维度堆叠成一个张量 \( X \in \mathbb{R}^{H \times W \times 3N} \)，而非独立编码。
  - 使用 VQ-VAE 将堆叠张量压缩到离散的视觉标记空间。编码器将 \( X \) 映射为连续隐变量 \( z_e \)，通过大小为 \( K \) 的码本（codebook）进行最近邻量化得到 \( z_q \)，解码器重建堆叠张量并拆分回各视角图像。
  - 损失函数：\( \mathcal{L}_{\text{VQ}} = \|X - \hat{X}\|_2^2 + \|\text{sg}[z_e] - z_q\|_2^2 + \beta \|z_e - \text{sg}[z_q]\|_2^2 \)，其中 \(\text{sg}\) 为停止梯度操作符。
  - 最高分辨率标记图为 \(16 \times 16\)，码本含 4096 个条目，每个码维度 32。一个标记代表一个多视角场景单元，而非单个图像块。
- **阶段二：文本条件多视角生成**
  - 生成器遵循逐尺度预测范式（VAR），预测多尺度标记金字塔 \( R = (r_1, \dots, r_K) \)：
    \[
    p(R \mid y) = \prod_{k=1}^{K} p(r_k \mid r_{<k}, y)
    \]
  - 使用 CLIP 文本编码器提取标记级序列特征 \( c_{\text{seq}} \) 和全局池化特征 \( c_g \)。
  - 全局语义通过自适应层归一化（AdaLN）注入：\( \text{AdaLN}(h, c_g) = \gamma(c_g) \odot \text{LN}(h) + \delta(c_g) \)。
  - 局部文本属性通过交叉注意力注入：\( \text{CrossAttn}(H, c_{\text{seq}}) = \text{Softmax}\left( \frac{(H W_Q)(c_{\text{seq}} W_K)^\top}{\sqrt{d}} \right) c_{\text{seq}} W_V \)。
  - 训练采用教师强制（teacher forcing），推理时使用无分类器引导（CFG）和 top-k/top-p 采样，最终标记图由冻结的 VQ 解码器还原为多视角图像。
- **阶段三：手物网格恢复**
  - 对生成的多视角图像进行 HSV 颜色分割，提取手和物体掩膜。
  - 使用 Stable Diffusion LoRA 修复模块，分别补全被物体遮挡的手部区域和被手遮挡的物体区域。
  - 物体分支：将补全后的物体多视角图送入 InstantMesh 进行初始物体网格重建，经网格修复和 SDF 滤波。
  - 手部分支：使用 OmniHands 风格的手部先验和参数化手部模型（MANO）获取手部网格。
  - 联合优化变量：\( \Xi = \{\Theta_h, T_h, R_o, t_o, \Delta M_o\} \)，包含手部参数、手部全局变换、物体旋转平移、物体顶点偏移。
  - 优化目标：
    \[
    \Xi^* = \arg\min_{\Xi} \lambda_{\text{mv}} \mathcal{L}_{\text{mv}} + \lambda_c \mathcal{L}_{\text{contact}} + \lambda_p \mathcal{L}_{\text{pen}} + \lambda_r \mathcal{L}_{\text{reg}}
    \]
    其中重投影和掩膜一致性项构成多视角损失，接触项为 \( \mathcal{L}_{\text{contact}} = \sum_{v \in C_h} \max(0, d(v, V_o) - \tau_c) \)，穿透项为 \( \mathcal{L}_{\text{pen}} = \sum_{v \in V_h} \max(0, -\phi_o(v)) \)（\(\phi_o\) 为物体有符号距离场）。
  - 优化分三步：全局多视角对齐 → 交互校正 → 局部细化与防穿透后处理。

## 3. 实验设计
- **数据集**：基于 HO3D 数据集构建，包含 YCB-Video 物体及配对的手/物体几何信息。用于表示和生成模块的训练帧共 16,291 张（训练 14,662，验证 1,629）。
- **渲染设定**：使用 PyTorch3D 以 256×256 分辨率渲染多视角图像，采用固定相机布局（方位角起始 30°，步长 60°；仰角在 20° 和 -10° 交替）共 6 个视角，类似于 Zero123++ 的轨道设置。
- **评估指标**：
  - 表示质量：PSNR、SSIM、LPIPS。
  - 物体几何：Chamfer Distance (CD) 和 F-score（阈值 5mm, 10mm），经尺度感知的 ICP 配准后在 10k 表面点上计算。
  - 手部精度：MPJPE（21 个关键点）、MPVPE（778 个 MANO 顶点），经 Procrustes 对齐。
  - 交互质量：穿透体积（PV）、穿透顶点占比（%PV）、平均/最大穿透深度、接触比率（距离阈值 2/5/10mm）。PV 通过 10 万点蒙特卡洛采样估计。
- **对比基线**：核心对比为单视角恢复 vs. 多视角恢复。单视角设置用 Zero123 将视角 0 扩展为多视角后再送入 InstantMesh，手部使用单视图先验；多视角直接使用固定视角图。所有权重、恢复模块、优化步骤和评估代码完全相同，仅输入视图数量不同。

## 6. 主要结论与发现
- 提出的多视角视觉 token 表示能够有效压缩手物场景结构，同时为生成提供离散词汇接口。
- 文本条件生成模块可产生语义对齐且跨视角一致的多视图手物图像。
- 在恢复阶段，**多视图输入相比单视图大幅提升了物体几何精度（CD 从 17.26mm 降至 4.92mm，F@10 从 70.6% 升至 98.6%）**，手部误差减半，更重要的是**穿透体积从 5.3721 cm³ 锐减至 0.2193 cm³**，同时保留了一定的接触比率，证明优化并非简单地推开手物，而是去除了严重穿透同时保持接触邻近性。
- 整体表明，**多视图视觉 token 是连接文本语义生成与几何恢复的有效中间表示**，解耦设计有助于故障定位。

## 7. 优点
- **解耦设计清晰**：将高维文本到复杂三维网格问题分解为易管理的子问题，每个阶段职责明确，便于诊断和优化。
- **紧凑表示**：堆叠多视角为单一张量并学习共享码本，既捕获跨视图相关性，又显著提升吞吐量（40 views/s vs. 6.7 views/s），避免序列处理。
- **强多视图监督**：恢复阶段充分利用固定多视角的联合优化和接触/穿透约束，针对性解决了单视图固有的几何歧义和穿透问题。
- **受控对比**：单/多视图消融实验设计严谨，变量控制严格，结论可信。

## 8. 不足与局限
- **评估规模**：定量恢复实验仅基于 9 个 HO3D 衍生样例，虽能证明多视图优势，但泛化性验证不足，缺少大规模或真实场景测试。
- **中间依赖**：生成质量高度依赖分割和修复模块的准确性，若多视角生成出现不一致或修复失败，会导致恢复误差传播。
- **未闭环训练**：当前恢复阶段的损失未反馈给生成器，3D 一致性约束仅在后处理优化中体现，生成模型本身不保证几何一致性。
- **缺少外部对比**：未与现有文本驱动手物生成方法（如 Text2HOI）或通用文本到 3D 方法在相同设定下直接比较，难以评估整体相对水平。
- **相机设定固定**：多视角生成和恢复均基于预设的固定相机布局，未展示对任意视角输入的灵活性。
- **应用限制**：方法输出静态网格，不涉及手物交互运动序列，对动态交互生成需单独扩展。
