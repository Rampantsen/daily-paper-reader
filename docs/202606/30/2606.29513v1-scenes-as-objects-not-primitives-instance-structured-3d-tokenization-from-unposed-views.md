## 摘要
三维场景应通过其包含的对象而非构成它们的基元来理解。然而，前馈重建方法输出的是密集、无结构的点或高斯体，对象级结构需事后恢复。我们提出一种前馈框架，直接从多视角未标定图像中将场景分解为实例结构化的三维标记组——这些紧凑且以对象为中心的单元可支撑后续的重建、分割与操作。每个标记组将捕获实体级身份的实例标记与编码局部几何和外观的锚定标记配对，进而解码为一组三维高斯体。这种两级分解将对象身份与局部外观解耦，使对象实例成为表示的原生接口而非衍生结果。标记组通过可微渲染在联合重建与分割监督下习得，无需任何三维标注。我们的前馈模型在类别无关的实例分割上超越了逐场景优化基线，同时在新视角合成上保持竞争力。除此之外，相同的标记组可直接解锁实例级场景编辑——通过对组进行操作来实现对象的移除、平移或插入——以及高效的开词汇三维实例检索，其中检索复杂度与实例数目而非基元数目成比例增长。

## Abstract
A 3D scene is understood through its objects, not the primitives that compose them. Yet feed-forward reconstruction methods output dense, unstructured sets of points or Gaussians, leaving object-level structure to be recovered after the fact. We propose a feed-forward framework that decomposes a scene into instance-structured 3D token groups directly from unposed multi-view images -- compact object-centric units from which reconstruction, segmentation, and manipulation all follow. Each token group pairs an instance token capturing entity-level identity with anchor tokens that encode local geometry and appearance, which are decoded into a set of 3D Gaussians. This two-level factorization decouples object identity from local appearance, making object instances a native interface of the representation rather than a derived product. The token groups are learned through differentiable rendering with joint reconstruction and segmentation supervision, requiring no 3D annotations. Our feed-forward model surpasses per-scene optimization baselines in class-agnostic instance segmentation while remaining competitive in novel view synthesis. Beyond these metrics, the same token groups directly unlock instance-level scene editing -- removing, translating, or inserting objects by operating on their groups -- as well as efficient open-vocabulary 3D instance retrieval, where retrieval complexity scales with the number of instances rather than primitives.

### 2. 方法论

论文提出 **实例结构化三维标记组（Instance‑Structured 3D Token Groups）**，将整个前馈重建、分组和语义提升统一在一个框架中。

**核心思想**  
- 从多视图图像中预测两种标记：**实例标记（instance token）** 概括对象的身份和范围；**锚定标记（anchor tokens）** 编码局部几何与外观，每个锚定标记可解码为一组三维高斯体。  
- 通过软分配（softmax 相似度）将每个锚点分配给一个实例标记，形成 L 个标记组，每组代表一个对象实例。  
- 这种两级分解使对象身份与局部细节解耦，实例成为表示的原生接口。

**关键技术细节**  
1. **多视图编码**：使用冻结的三维基础模型（VGGT）提取特征和点图，经 patchify 并与 RGB、点图融合后形成上下文特征 X。  
2. **标记初始化和解码**：  
   - 从点图中进行最远点采样得到 K 个锚点位置，初始锚点标记 A^(0) = 上下文特征 + 位置嵌入。  
   - 实例标记 G^(0) 为可学习嵌入（L=100）。  
   - 两个交叉注意力变换器：**图像‑锚点解码器 D_anchor** 将锚点标记与 X 交互更新为 A；**锚点‑分组解码器 D_group** 将实例标记与 A 交互更新为 G。  
3. **锚点到组的分配**：π_{k,ℓ} = softmax(⟨A_k, G_ℓ⟩)_{ℓ}，带一个额外的 void 通道以处理无实例区域。  
4. **高斯重建**：每个锚点 A_k 通过 MLP 预测 N_g=32 个高斯体（位置偏移、尺度、旋转、透明度、SH 系数），每个高斯继承其锚点的分配概率 π，从而可独立渲染各实例。  
5. **训练监督**：  
   - **重建损失** L_render: MSE + LPIPS。  
   - **分组损失** L_seg: 将渲染的实例概率图与真实 2D 实例掩码进行匈牙利匹配后计算 BCE + Dice 损失，并采用 λ_seg 预热策略。  
6. **语义特征分解蒸馏**：  
   - 训练另一个锚点到组的交叉注意力模块，从二维基础模型（如 LSeg）特征中提取组级语义嵌入 s_ℓ（512维）和锚点级残差 r_k（8维）。  
   - 渲染时重构完整特征： F_v(u) = Σ_ℓ Ŝ_{v,ℓ}(u) s_ℓ + W_r R̂_v(u)。  
   - 损失包括全像素特征对齐损失和组级平均特征对齐损失。

### 6. 主要结论与发现

- 所提的实例结构化标记组从多视图图像中直接习得实例级场景表示，无需三维标注。  
- 在类别无关实例分割上显著超越所有基线，包括逐场景优化的方法。  
- 重建质量与像素级方法（Uni3R）接近，但表示更紧凑（语义存储仅为 59.4K vs 8.4M）。  
- 语义分解提升优于基线，且由于组级特征共享，检索效率高、不依赖后处理。  
- 联合训练与预热策略对形成稳定分组至关重要（顺序训练导致 AP 骤降）。  
- 跨数据集泛化（RealEstate10K 使用 SAM2 伪标签，MipNeRF360 零样本）表明方法具有一定鲁棒性。

### 7. 优点

- **原生对象级界面**：首次将实例作为前馈重建的显式单元，而非后处理，极大简化下游应用（编辑、检索）。  
- **紧凑高效**：少量标记组（≤100）替代数万高斯体进行语义存储和操作，复杂度与实例数线性相关。  
- **无需三维标注**：完全通过二维监督（RGB + 实例掩码）训练，降低了数据需求。  
- **灵活的应用接口**：相同的标记组既可用于重建、分割，又可直接支持实例级编辑和开放词汇检索。  
- **可微渲染与联合训练**：通过可微渲染和软分配实现端到端学习，训练过程稳定（预热策略）。

### 8. 不足与局限

- **场景尺度限制**：主要针对室内有限场景评估，最大组数固定为 L=100，室外或大规模场景需重新设计。  
- **静态假设**：当前方法假设静态场景，动态物体（如机器人操作）未涉及。  
- **表示容量**：单一组级共享特征可能不足以描述高度复杂或纹理差异大的实例，可能需要多个共享基特征。  
- **重建质量稍逊**：由于使用了紧凑的标记表示，PSNR 和 SSIM 略低于像素对齐的高斯方法，细节有时被平滑。  
- **伪标签依赖**：RealEstate10K 实验使用 SAM2 自动掩码，掩码 ID 断裂可能影响分组一致性（但论文仍取得不错结果）。  
- **跨域泛化**：虽然展示了零样本迁移，但在 MipNeRF360 上的绝对重建指标仍较低（16.52 PSNR），未与微调等方法对比。
