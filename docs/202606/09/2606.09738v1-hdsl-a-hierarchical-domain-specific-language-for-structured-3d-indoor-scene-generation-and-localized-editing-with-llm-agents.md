## 摘要
文本驱动的室内场景生成与编辑需要一种语言模型既能生成也能修改的中间表示。现有基于LLM的系统通常依赖场景图或全局约束列表，这些表示虽然紧凑，但对局部几何信息描述不足，使得基于指令的编辑难以定位。我们将此问题定义为结构化程序生成与局部程序修复，并提出层次化描述场景语言(HDSL)，一种类似XML/CSS的领域特定语言，用于结构化三维室内场景。HDSL将房间、区域、物体和支撑面表示为带有局部坐标的树状结构，使得复杂场景更易于递归规划，也更易于检索以进行编辑。我们的流水线使用LLM智能体生成经过有界验证的HDSL子树，通过多模态资源检索将非虚拟节点实例化，并应用力导向布局优化来修复边界和碰撞错误。在编辑方面，层次化检索增强生成(HRAG)检索相关子树，要求LLM仅重写该局部上下文，并通过确定性的三方合并将结果合并回去。在我们复现的基准测试中，HDSL在平均物体覆盖率、文本-场景对齐度和生成时间方面均优于完整的文本到场景基线方法，同时在几何指标上与最近的仅布局重建方法保持竞争力；在编辑方面，HRAG将令牌使用量降低了5.22倍，运行时间缩短了6.19倍，为所有八项配对编辑生成有效的DSL，并更好地保留了不相关的场景物体。

## Abstract
Text-driven indoor scene generation and editing require an intermediate representation that language models can both produce and revise. Existing LLM-based systems often rely on scene graphs or global constraint lists, which are compact but underspecify local geometry and make instruction-based edits difficult to localize. We frame this problem as structured program generation and local program repair, and propose Hierarchical Descriptive Scene Language (HDSL), an XML/CSS-style domain-specific language for structured 3D indoor scenes. HDSL represents rooms, regions, objects, and support surfaces as a tree with local coordinates, making complex scenes easier to plan recursively and easier to retrieve for editing. Our pipeline uses LLM agents to generate HDSL subtrees with bounded verification, grounds non-virtual nodes through multimodal asset retrieval, and applies force-directed layout optimization to repair boundary and collision errors. For editing, Hierarchical Retrieval-Augmented Generation retrieves the relevant subtree, asks the LLM to rewrite only that local context, and merges the result back through a deterministic three-way merge. In our reproduced benchmark, HDSL improves average object coverage, text-scene alignment, and generation time over full text-to-scene baselines while remaining competitive with recent layout-only reproductions on geometry metrics; for editing, HRAG reduces token use by $5.22\times$ and runtime by $6.19\times$, produces valid DSL for all eight paired edits, and better preserves unrelated scene objects.

#### 2.2 层次化生成流水线
- **递归生成**：LLM 智能体以当前节点的祖先路径、父几何信息、全局指令等为上下文，逐步生成直接子节点（如房间→功能区→家具组→具体物体），每次只输出一个可解析的 XML 片段。
- **生成–验证–修正循环**：采样多个候选 DSL 片段，进行 XML 解析、结构/空间有效性检查（如边界合规性、遮挡、重复标识等），对可确定性修复的错误进行修正，否则拒绝并反馈给 LLM 重试。
- **空间协调**：同一父容器下的兄弟节点在一次 LLM 调用中联合生成以保证相对位置协调；仅有不相交的子树可并行展开。
- **虚拟容器与叶节点判定**：低于面积阈值或深度超过最大展开深度的节点视为叶节点，避免过度细分。
- **术语**：虚拟节点（区域或支撑容器）不进行资产检索，非虚拟节点才进行检索。

#### 2.3 多模态资产检索
- 对非虚拟节点使用 Objaverse 资产库和 Holodeck 检索流程，结合 OpenCLIP 图文相似度、Sentence-BERT 文本相似度、类别约束和物理尺寸匹配，为每个节点选取最匹配的 3D 模型。

#### 2.4 力导向布局优化 (FDLO)
- **目的**：修复 LLM 生成布局中可能存在的边界侵犯、兄弟重叠、容器过紧或朝向错误。
- **核心公式**（文本描述）：
  - 边界吸引力：\( \vec{F}_{a} = \alpha \frac{\vec{D}_i}{\|\vec{D}_i\|} \)，其中 \(\vec{D}_i\) 指向有效父区域内部。
  - 重叠排斥力：\( \vec{F}_{ij}^r = \beta \cdot \text{IoU}_{ij} \cdot \frac{\vec{\Delta}_{ij}}{\|\vec{\Delta}_{ij}\|} \)，其中 \(\vec{\Delta}_{ij}\) 分离重叠兄弟，IoU 为 3D 边界框交并比。
  - 位置更新：\( p_i^{t+1} = p_i^t + \vec{F}_{a} + \sum_{j\neq i} \vec{F}_{ij}^r \)。
- 实际流程包括自适应力缩放、边界钳位、多阶段修复和自动墙面朝向校正。编辑时优化仅限制于可移动节点，以保持无关区域不变。

#### 2.5 层次化检索增强生成 (HRAG) 局部编辑
- **检索**：对每个节点拼接标识符、类别、描述和祖先路径形成文本特征，采用 BGE 族编码器计算与编辑指令的嵌入相似度，结合词汇匹配和意图感知约束（如重排编辑保留更宽上下文，移除/替换则缩小目标集）。
- **选点公式**：选择满足 \( s_{\text{max}} - s_i \leq \delta_e \) 或关键词奖励 \( b_i > 0 \) 的节点，再取前 K 个构成相关节点集 \( N_{\text{rel}} \)，并向上、横向扩展以包含祖先和兄弟节点，形成检索到的局部子树 \( \Phi_{\text{rag}}(S) \)。
- **LLM 局部重写**：将检索到的子树提供给 LLM，要求返回同一个根标识的完整编辑后片段，并保留不相关的兄弟节点及资产元数据。
- **确定性三方合并**（图4）：
  - 基于原始树 \( \Phi(S) \)、未编辑的检索子树 \( \Phi_{\text{rag}}(S) \) 和编辑子树 \( \Phi'_{\text{rag}}(S) \) 进行合并。
  - 规则：仅对选定目标子树应用编辑属性；上下文节点保留原始属性；仅在移除/替换意图下允许删除；为未改变对象继承资产元数据；新增子节点仅在添加/替换意图下接受。
