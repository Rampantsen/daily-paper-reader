## 摘要
3D高斯泼溅（3DGS）能够实现高保真、实时的3D场景重建，但将训练扩展到大规模场景时，需要跨多个GPU优化数亿个高斯。现有分布式方法要么将场景划分为孤立区域，导致全局不一致，要么依赖全局高斯层面的交换，这会使得GPU间通信量大幅增长，并迅速主导迭代时间。我们提出Splaxel，一个基于像素级局部渲染和全局合成的通信高效分布式3DGS训练框架。该框架不进行高斯同步，而是由每个GPU渲染其局部子集并仅交换部分像素值，在保持数学一致性的同时使通信成本随场景规模增长而保持稳定。Splaxel进一步通过几何和透射可见性预测减少像素级冗余，并通过无冲突的相机视图整合提升GPU利用率。在包含高达1.2亿个高斯的大规模数据集上评估，Splaxel相较于最先进的分布式3DGS框架实现了最高7.6$\times$的加速，同时保持了高重建质量。

## Abstract
3D Gaussian Splatting (3DGS) enables high-fidelity and real-time 3D scene reconstruction, but scaling training to large-scale scenes requires optimizing hundreds of millions of Gaussians across multiple GPUs. Existing distributed approaches either partition scenes into isolated regions, causing global inconsistency, or rely on global Gaussian-level exchanges, which lead to substantial growth in inter-GPU communication and quickly dominate iteration time.   We propose Splaxel, a communication-efficient distributed 3DGS training framework based on pixel-level local rendering and global composition. Instead of synchronizing Gaussians, each GPU renders its local subset and exchanges only partial pixel values, maintaining mathematical consistency while keeping communication cost stable as the scene size increases. Splaxel further reduces pixel-level redundancy through geometric and transmittance visibility prediction and improves GPU utilization via conflict-free camera-view consolidation. Evaluated on large-scale datasets with up to 120M Gaussians, Splaxel achieves up to 7.6$\times$ speedup over the state-of-the-art distributed 3DGS framework while preserving high reconstruction quality.

## 1. 研究背景与核心问题

- **核心问题**：3D Gaussian Splatting (3DGS) 在大规模场景重建中，需要优化数亿个高斯体，单个 GPU 内存和算力不足，必须使用分布式训练。
- **现有方法局限**：
  - **基于算法优化的分治法**：将场景划分为独立区域，各 GPU 单独训练后合并，虽然能适应视角变化，但牺牲了全局一致性，导致重建结果出现明显拼接痕迹。
  - **基于系统优化的全局高斯通信（如 Grendel）**：保持全局一致性，但每个 GPU 需要根据当前相机视角向其他 GPU 请求所有缺失的高斯体，这种**高斯级别的全交换（all-to-all）通信成本随场景规模（高斯数量）和 GPU 数量非线性增长**，在大规模场景中通信时间占比可超过 70%，成为训练瓶颈。
- **核心矛盾**：如何在保持全局一致重建质量的前提下，彻底打破通信开销随场景规模膨胀的限制。

## 2. 方法论

### 2.1 核心思想：像素级通信范式
- 将分布式训练从“交换高斯体”转为“交换部分像素（像素片段）”。
- 每个 GPU 对自己的高斯体分区进行局部渲染，生成部分像素值（颜色、透射率、深度），然后在 GPU 间仅交换这些中间像素片段，最后进行全局合成并回传梯度。
- **通信成本恒定**：通信量只与渲染分辨率有关，与高斯体总数无关。

### 2.2 保持全局顺序：凸分区与像素级全局合成
- **挑战**：局部渲染丢失了全局高斯体的顺序信息，直接混合易导致穿模。
- **解决方案**：采用**凸分区**（AABB 轴对齐包围盒），保证从相机出发的每条光线与任意 GPU 分区的交**最多一次**。
- **算法流程**（图9）：
  1. **局部渲染**：GPU $m$ 为其局部高斯集 $\mathcal{Q}_m$ 计算部分像素颜色 $C_p^m$、部分透射率 $T_p^m$ 和部分深度 $D_p^m$。
  2. **全局合成**：按 $D_p^m$ 排序 GPU 片段，从前向后累加得到最终像素颜色：
     $$C_p = \sum_{m} C_p^m \prod_{k<m} T_p^k$$
  3. **反向传播**：利用通信中已获得的 $C_p^m$ 和 $T_p^m$，各 GPU 可本地完成精确梯度计算，无需额外通信。

### 2.3 像素级冗余消除
- **空间冗余（几何不可见）**：相机视锥与部分 GPU 分区无交集，这些 GPU 产生的像素全部为零。
  - **方法**：利用视锥和凸分区的交集为凸多面体，将其顶点投影至图像平面，生成可见像素区域图，仅对此区域内像素进行选择性的 all-to-all 通信，跳过不可见像素。
- **饱和冗余（透射率不可见）**：训练中前方高斯体变得足够不透明时，后方的贡献完全消失。
  - **方法**：在全局合成过程中实时检测累计透射率，一旦低于阈值 $\varepsilon$，则将后方 GPU 对应像素标记为无效，后续迭代中不再传输和计算。

### 2.4 GPU 利用率优化：无冲突视图合并
- **问题**：单视图训练时，该视图可能只覆盖少数 GPU，其它 GPU 空闲。
- **方法**：分析所有视图所需的参与 GPU 集合，将无交集（无冲突）的视图合并到同一批次同时执行。采用贪心桶装法进行增量合并，并异步重叠执行，有效减少 GPU 空闲气泡，提升整体吞吐。

### 2.5 其他关键技术
- **跨边界高斯体处理**：对于边界处可能引起全局顺序交错的高斯体，通过深度和可见区域联合判别，进行保守剔除，保证数学一致性。
- **动态重分区**：在训练中监测负载均衡，不均衡超过阈值时用 KD-tree 重分区，开销极低（<0.7% 训练时间）。该设计增强了鲁棒性。

## 6. 主要结论与发现
- **加速显著**：在 8 GPU 上，Splaxel 相较于 Grendel 在四个大规模数据集上达到 2.9×–7.6× 的训练加速。
- **通信瓶颈解除**：像素级通信使通信量恒定，通信时间占比降至 10%–20%，不再主导训练周期。
- **质量保持甚至提升**：在取得大幅加速的同时，PSNR 与 Grendel 持平或略优（0.2–0.9 dB 提升），且避免了区域拼接带来的不一致性。

- **可扩展性强**：训练吞吐量随 GPU 数量近乎线性增长，而 Grendel 在超过 4 GPU 后加速有限。

## 7. 优点与亮点
- **通信恒定**：将分布式训练从高斯数量依赖中解耦，根本性解决了大规模场景的通信爆炸问题。
- **数学严格性**：通过凸分区保证全局合成时的深度顺序，使局部渲染与全局渲染等价，训练信号无损。
- **冗余消除创新**：静态几何预测与动态透射率检测相结合，在无额外通信开销下大幅减少无效传输和计算。
- **系统优化周全**：无冲突视图合并、跨边界高斯体处理、自适应重分区，都体现了对实际训练动态性的细致考量。
- **实验充分**：覆盖不同规模、不同视角（街道/航空）的场景，且进行了多维度消融与稳定性分析，验证了方法的有效性和鲁棒性。

## 8. 不足与局限
- **分区形状限制**：依赖轴对齐凸分区，对于某些极度细长或非轴对齐的场景可能造成负载不均，文中通过 KD-tree 和动态重分区缓解，但非凸拓扑仍可能受限。
- **对训练前期假设敏感**：饱和冗余依赖于训练过程中不透明度逐渐增高，在训练早期效果有限，但在整体训练周期中平均加速仍显著。
- **未对比其他分布式框架**：主要基线仅 Grendel，未与其它基于分治的分布式 3DGS 方法（如 DOGS、VastGaussian）进行直接性能-质量权衡对比，仅指出其全局不一致性。
