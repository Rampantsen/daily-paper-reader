## 摘要
开放词汇3D功能性分割使机器人能够在3D场景中定位功能性物体组件。这是一项具有挑战性的任务，需要空间理解和任务解读。当前的开放词汇3D分割方法主要关注物体级别的识别，而场景范围的部分分割方法则试图彻底分割整个场景，这使得它们非常消耗资源且耗时。在分割粒度、精度和速度之间取得平衡仍然是一个挑战。为了缓解这一问题，我们提出了T-FunS3D，一种任务驱动的分层开放词汇3D功能性分割方法，为机器人应用提供可操作的感知。我们的方法以室内场景的3D点云和带姿态的RGB-D图像作为输入。我们通过提取环境中的实例及其视觉嵌入来构建开放词汇场景图。给定任务描述，T-FunS3D识别场景图中最相关的实例，并利用视觉-语言模型定位其功能性组件。在SceneFun3D数据集上的实验表明，T-FunS3D在开放词汇3D功能性分割方面与现有技术相当，同时实现了更快的运行速度和更低的内存占用。

## Abstract
Open-vocabulary 3D functionality segmentation enables robots to localize functional object components in 3D scenes. It is a challenging task that requires spatial understanding and task interpretation. Current open-vocabulary 3D segmentation methods primarily focus on object-level recognition, while scene-wide part segmentation methods attempt to segment the entire scene exhaustively, making them highly resource-intensive and time consuming. Balancing segmentation performance in terms of granularity, accuracy, and speed remains a challenge. As one step towards alleviating this, we introduce T-FunS3D, a task-driven hierarchical open-vocabulary 3D functionality segmentation method that provides actionable perception for robotic applications. Our method takes as input the 3D point cloud and posed RGB-D images of an indoor scene. We construct an open-vocabulary scene graph by extracting instances and their visual embeddings in the environment. Given a task description, T-FunS3D identifies the most relevant instances in the scene graph and locates their functional components leveraging a vision-language model. Experiments on the SceneFun3D dataset demonstrate that T-FunS3D is comparable to state-of-the-art in open-vocabulary 3D functionality segmentation, while achieving faster runtime and reduced memory usage.

- **研究背景与动机**：

    - 因此，该论文旨在提出一种**任务驱动**的分层方法，仅对与任务相关的对象进行精细分割，以在**分割粒度、准确性和推理速度**之间寻找更优的平衡。

### **2. 方法论**

- **核心思想**：T-FunS3D是一个无训练的、分两阶段运行的流水线方法。
    - **第一阶段（场景理解）**：预先构建一个轻量级的开放词汇3D场景图，为整个环境建立语义索引。
    - **第二阶段（任务执行）**：收到任务后，首先解析任务，然后查询场景图定位相关物体，最后仅对这些物体进行功能部件的精细分割。
- **关键技术细节与流程**：
    1.  **开放词汇3D实例分割**：
        - 利用预训练模型Mask3D进行**类别无关的实例分割**。
        - 为每个实例从多视角RGB图像中提取**视觉语义嵌入**。这里采用了**多尺度裁剪与遮罩裁剪**相结合的策略来增强嵌入的鲁棒性，避免前景-背景偏差。
        - 该视觉嵌入由微调过的CLIP模型（FG-CLIP）计算得出。
    2.  **开放词汇场景图构建（第一阶段）**：
        - 将上一步得到的每个3D实例作为图的**节点**，并存储其平均视觉嵌入。
        - **边**不进行预计算和存储，而是在查询时动态构建：通过计算两个候选节点各自全尺寸图像视觉嵌入的平均值，来编码节点间的空间语义关系。
    3.  **大语言模型（LLM）任务理解（第二阶段）**：
        - 收到自由形式任务查询 \( Q \) 后，利用LLM（Qwen3-14B）将其解析为任务本体（JSON格式），提取出：
            - **上下文对象（C）**：机器人需交互的主体，如“抽屉”。
            - **功能组件（F）**：要操作的部分，如“把手”。
            - **空间关系（S）**与**参照物（R）**：“右边”（S）和“床”（R）。
    4.  **上下文对象定位**：
        - 将文本形式的\( C \)、\( R \)、\( S \) 编码为文本嵌入，在场景图中搜索匹配的节点。
        - 通过计算文本嵌入与节点视觉嵌入的**余弦相似度**，找出候选的\( C \)和\( R \)节点。
        - 动态计算候选\( C \)和\( R \)节点之间的边嵌入，并与文本化的空间关系\( S \)进行相似度匹配，最终唯一确定目标实例。
    5.  **功能性组件分割**：
        - 定位到目标上下文对象后，将其最具可见性的2D图像视图送入视觉-语言模型Molmo。
        - Molmo根据任务本体提示，输出功能组件的像素坐标。
        - 这些坐标作为SAM的提示点，生成2D分割蒙版。
        - **关键设计选择**：选择生成的蒙版中**最小的一个**，而非置信度最高的一个，以避免过低分辨率裁剪下的过分割（分割出整个物体）。
        - 将该2D蒙版反投影到3D空间，并进行多视角聚合，得到最终的3D功能性分割结果。
