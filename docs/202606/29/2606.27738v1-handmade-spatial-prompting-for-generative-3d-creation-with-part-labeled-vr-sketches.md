## 摘要
文本到3D生成降低了3D内容创作的门槛，但纯文本在指定空间意图方面是一种弱的界面：部件应该放在哪里、它们如何关联、以及物体在3D中应该如何组织。我们提出了HandMade，一种结合VR 3D草图绘制和语言的工作流，用于开放域的3D资产生成。HandMade将粗略的、带部位标注的3D草图视为现有生成模型的空间提示，而不是直接重建的不完整几何体。它将分段的VR笔触转换为多视图部位引导和结构化提示，使用户能够通过3D草图绘制指定对象布局和部件关系，同时使用语言来指定标识、材质、风格和局部细节。技术评估显示，在20个多样化示例上，HandMade比纯文本和基于草图的基线方法更好地保留了用户创作的空间支架。一项有八名参与者参与的用户研究，描述了用户如何在初始创作和后续修订过程中利用3D草图绘制来定义空间布局，以及利用语言来定义标识、材质和细节。HandMade贡献了一种交互范式和从界面到生成的流程，用于空间引导的3D创作。

## Abstract
Text-to-3D generation lowers the barrier to 3D content creation, but text alone is a weak interface for specifying spatial intent: where parts should be placed, how they relate, and how an object should be organized in 3D. We present HandMade, a workflow that combines VR 3D sketching and language for open-domain 3D asset generation. HandMade treats coarse, part-labeled 3D sketches not as incomplete geometry to reconstruct directly, but as spatial prompts for existing generative models. It converts segmented VR strokes into multi-view part guidance and structured prompts, allowing users to specify object layout and part relationships through 3D sketching while using language for identity, material, style, and local details. A technical evaluation shows that HandMade better preserves user-authored spatial scaffolds than text-only and sketch-based baselines on 20 varied examples. A user study with eight participants characterizes how users make use of 3D sketching for spatial layout and language for identity, materials, and details across initial authoring and subsequent revision. HandMade contributes an interaction paradigm and interface-to-generation pipeline for spatially guided 3D creation.

## 1. 论文的核心问题与整体含义
- **核心问题**：文本到3D生成使得3D内容创作更加容易，但纯文本是一种很弱的界面来表达**空间意图**——部件应该放在哪里、它们的相对大小、如何连接、以及如何在3D空间中组织对象。此外，用文字编辑或修改3D模型也十分笨拙。
- **整体含义**：论文提出了一种**空间提示**（Spatial Prompting）交互范式，将用户粗略的、带部位标签的VR 3D手绘草图作为生成模型的空间引导，而非直接重建的目标几何。通过将粗糙草图与语言描述相结合，HandMade使非专业用户也能通过绘图表达空间布局，用语言指定物体身份、材质和风格，从而实现开放域3D资产生成。

## 2. 方法论：核心思想与关键技术细节
- **核心思想**：将VR 3D草图视为**空间提示**，而非精确几何约束。系统将分段的VR笔触转化为多视图部位引导图，并结合用户提供的文本描述生成结构化提示，驱动现有的图像生成和图像到3D模型。
- **工作流程（Pipeline）**：
    - **输入**：用户在VR中分部绘制粗糙的3D笔触，每个部分用一种系统分配的独特颜色标注，并键入部件名称和可选的部件描述，外加一个对象级别的文本描述。
    - **多视图部位引导**：将归一化到单位立方体中的3D草图从4个固定视角（前、右、后、左，零仰角）渲染为2×2的引导图，每个部件以其引导色绘制在白色背景上。
    - **结构化提示构建与多视图图像合成**：将引导图与一个结构化文本提示（包含对引导图应视为粗略空间表达的指示、颜色-部件映射、用户的对象及部件描述）一起送入图像生成后端（ChatGPT Images 2.0 API），生成干净的多视图对象图像。
    - **3D形状生成**：将生成的2×2多视图图像分割成四个视图，送入多视图图像到3D重建后端（Hunyuan3D-2mv），生成带纹理的网格模型。
    - **修订**：用户可以基于中间多视图图像或最终网格，修订VR草图或文本描述，或者重新运行生成阶段。
- **原型实现**：VR草图绘制使用Meta Quest Pro头显，手部追踪进行绘图，另一只手用物理键盘输入文本和切换模式。图像生成使用ChatGPT Images 2.0的编辑接口，3D重建使用Hunyuan3D-2mv（形状生成约17秒，纹理生成约28秒，总计约45秒）。

## 6. 主要结论与发现
- **空间引导有效**：HandMade在所有几何对齐指标上均优于纯文本和单视图消融，也优于其他草图基线（Sketch→Mesh 0.035，F-score 0.727）。3D草图提供了文本难以表达的空间结构信息。
- **多视图重建有益**：使用多视图引导比单视图能更好地保留跨视图的部件关系和空间结构。
- **粗草图可作为空间提示**：VR手绘的粗糙、不完整、不精确的笔触可以被现有生成模型用作空间提示，不需要完美几何。
- **用户分工**：用户使用3D草图表达部件的空间布局和关系，使用语言描述物体身份、材质、风格和细节。
- **修订支持**：生成中的中间产物（多视图图像、网格）充当检查点，使用户可以选择性修改文本、重新生成或调整草图，大多数修订无需重新绘制VR草图。

## 7. 论文优点
- **创新交互范式**：提出空间提示的概念，将粗略3D草图作为生成模型的引导而非直接重建，降低了使用门槛。
- **多模态互补**：清晰划分3D草图（空间布局）和语言（语义/外观）的分工，符合认知习惯。
- **模块化设计**：利用现有生成模型作为后端，无需训练专用草图到3D模型，保持了开放域能力。
- **支持迭代修订**：提供多级检查点和修订路径，符合生成式AI中的metacognitive过程。
- **技术评估全面**：包含多种基线和消融，定量指标明确，并配有用户研究分析交互行为。

## 8. 不足与局限
- **输入精度限制**：VR手绘受制于跟踪噪声、深度控制难和绘图技巧，无法表达精确几何、拓扑、对称性或部件间精确接触关系。
- **生成失败模式**：多视图图像生成可能忽略引导色、扭曲比例、产生视图不一致；3D重建可能将薄结构平滑、合并相邻部件或丢失空洞。
- **评价覆盖有限**：技术评估仅测量几何保持，未涉及美学、语义正确性或下游可用性；用户研究为描述性，样本量小，非对比性。
- **接口负担**：VR绘图界面带来操作负担（如键盘快捷键记忆、模式切换、视角移动），部分用户报告了眩晕和空间感不足。
- **过渡手动**：原型中的阶段过渡仍需研究者介入，未实现全自动循环。
- **基线限制**：比较限于有公开代码且输入可适配的方法，未能涵盖所有相关草图到3D方法。
- **应用限制**：主要面向早期概念创建，缺少生产级重拓扑和精确尺寸控制，后续专业编辑需借助传统工具。
