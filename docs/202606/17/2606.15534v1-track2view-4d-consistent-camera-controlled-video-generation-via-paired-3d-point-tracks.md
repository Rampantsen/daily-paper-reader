## 摘要
从新的相机视角重新渲染现有视频，要求输出遵循指定的相机轨迹，同时在每一帧中保留原始场景的外观和动态。现有方法依赖于逐帧姿态嵌入、带噪声的点云渲染或隐式学习的对应关系，这些都没有在源像素和目标像素之间提供显式、时间连续的链接。我们提出Track2View，它基于配对的3D点轨迹来调节视频扩散Transformer：将场景点投影到源相机视图和目标相机视图中的稀疏轨迹。这些轨迹提供了显式的时空对应关系，其本身在时间上是连续的，编码了什么内容应该出现在何处以及何时出现。Track2View的核心是一个双视图轨迹调节器，它通过无参数的几何操作和学习的时间聚合，将视觉上下文从源视图传输到目标视图，确保泛化到任意相机轨迹，而无需记忆特定的运动。我们进一步引入了一个数据整理流程，通过在时间上拼接的多相机视图对上运行3D点跟踪器来提取一对一的轨迹对应关系。在包含静态和动态场景的400个视频基准上，Track2View在视觉质量、视图同步和相机准确性方面都达到了最新水平，与领先的基线方法相比，旋转误差降低了30-65%，平移误差降低了61-72%。项目页面位于：https://qjizhi.github.io/track2view。

## Abstract
Re-rendering an existing video from a novel camera viewpoint requires the output to follow the prescribed camera trajectory while preserving the appearance and dynamics of the original scene across every frame. Existing methods rely on per-frame pose embeddings, noisy point-cloud renderings, or implicit learned correspondences, none of which provides an explicit, temporally continuous link between source and target pixels. We propose Track2View, which conditions a video diffusion transformer on paired 3D point tracks: sparse trajectories of scene points projected into both the source and target camera views. These tracks provide explicit spatiotemporal correspondences that are temporally continuous by construction, encoding what content should appear where and when. At the core of Track2View is a dual-view track conditioner that transfers visual context from source to target view through parameter-free geometric operations and learned temporal aggregation, ensuring generalization to arbitrary camera trajectories without memorizing specific motions. We further introduce a data curation pipeline that extracts one-to-one track correspondences by running a 3D point tracker on temporally concatenated multi-camera view pairs. On a 400-video benchmark spanning static and dynamic scenes, Track2View achieves state-of-the-art results across visual quality, view synchronization, and camera accuracy, reducing rotation error by 30-65% and translation error by 61-72% relative to leading baselines. Project page is available at this https URL: https://qjizhi.github.io/track2view

### 1. 论文的核心问题与整体含义（研究动机和背景）
- **核心问题**：现有的视频视角重渲染（video-to-video re-rendering）方法难以保证输出视频在任意目标相机轨迹下的**4D一致性**（即同时保持空间与时间上的连贯）。已有的三种策略都存在缺陷：
  - 基于姿态嵌入的方法通常假设源相机静止，丢弃源视频的时间上下文，且逐帧独立计算缺乏时间连续性。
  - 基于点云渲染的方法受噪声深度估计影响，且逐帧渲染丢弃了时间相关性，易产生闪烁。
  - 隐式场景传输方法将空间一致性完全交给模型注意力，缺乏显式的几何对应信号，大视角变化下相机精度下降。
- **整体含义**：论文提出 **Track2View**，通过**配对的3D点轨迹（paired 3D point tracks）** 为源视图和目标视图之间提供显式、时间连续的时空对应关系，从而在重新渲染时精确控制相机轨迹，并保持场景外观与动态的忠实还原。

### 2. 论文提出的方法论
- **核心思想**：利用3D点跟踪器（SpatialTrackerV2）从源视频中估计3D点轨迹，将其重投影到目标相机下，得到源和目标视图下**一一对应的2D轨迹坐标与深度**。这些轨迹作为扩散模型的条件，天然编码了“什么内容应该出现在何时何处”。
- **框架流程**：
  - 源视频经3D VAE编码得到潜在特征，并patch化为token。
  - 噪声目标潜在特征同样patch化，与源token沿帧维度拼接。
  - 双视图轨迹调节器基于配对的轨迹坐标与深度生成轨迹token，**逐元素加**到视频token上。
  - 对预训练的DiT（WAN-2.1）使用LoRA微调，仅去噪目标部分，保持基座模型冻结。
- **双视图轨迹调节器**：
  - **无参数几何操作**：通过双线性网格采样从源视频token中提取轨迹特征；用相同坐标和权重将聚合后的特征散射回密集网格。
  - **时间聚合 (Temporal Aggregation)**：使用Transformer编码器（默认8层），在轨迹维度聚合跨帧的视觉上下文，受遮挡或出界点通过填充掩码处理。
  - **深度编码**：对每个视图的逆深度进行傅里叶位置编码，与共享的时间特征相加，以区分视点并推理视差与遮挡。
- **训练数据整理**：
  - 从多相机同步视频（MultiCamVideo）中，将相机A的倒放视频与相机B拼接，形成一个首帧相同的长序列。
  - 在拼接序列上运行SpatialTrackerV2，查询点仅放置在前80帧（相当于源视频范围），跟踪器自然给出跨越源和目标段的匹配轨迹。
  - 将前81帧轨迹倒序恢复，得到成对的 `(T_src, T_tgt)`。
- **训练细节**：源潜在保持清洁，噪声仅加到目标潜在；使用流匹配损失；对源帧进行最高50%的随机丢弃，轨迹保留50-100%以提升鲁棒性。

### 3. 实验设计
- **数据集与基准**：
  - 使用 **RealCam-Vid** 基准，包含400个视频：200个静态场景（RealEstate10K）和200个动态场景（MiraData）。
  - 评价指标覆盖三大轴：视觉质量（FID↓, CLIP-T↑, CLIP-F↑）、视图同步（CLIP-V↑, Mat.Pix.↑）、相机准确性（RotErr↓, TransErr↓, 通过COLMAP/GLOMAP提取生成视频的相机位姿计算），以及VBench的六项感知/时间质量指标。
- **对比方法**：
  - 包括四个SOTA方法：**Trajectory Attention**、**TrajectoryCrafter**、**Gen3C**、**ReCamMaster**。前两者在I2V模式下评估相机精度（相对源帧），后两者在V2V模式下比较。评估时匹配不同输出帧数（25/49/81帧）。
- **主要结果**：
  - 所有帧数下，Track2View在FID、Mat.Pix、RotErr、TransErr等核心指标上均达到最优。
  - 81帧时：TransErr降61%（0.818 vs ReCamMaster的2.096），RotErr降30%；25帧时RotErr降65%（1.24° vs 3.54°）。
  - VBench指标一致领先或次优，证明控制精度提升未牺牲感知质量。
- **消融实验**：
  - 查询策略：在首个帧和中间帧同时查询（All，1152条轨迹）获得了最佳的Mat.Pix和RotErr，比仅首帧查询能覆盖场景新出现区域。
  - 时间聚合深度：2层到8层，RotErr从2.24°降至1.55°，TransErr从1.681降至0.818，表明深层聚合有效传递跨帧信息。

### 6. 论文的主要结论与发现
- 配对3D点轨迹能够提供**显式且时间连续的时空对应**，这是现有姿态嵌入和渲染代理所缺失的关键信号。
- 无参数的几何操作（采样/散射）使模型能够泛化到训练时未见过的任意相机轨迹，而学习的时序聚合有效传播了遮挡区域和长距离的视觉信息。
- Track2View在视觉质量、视图同步和相机精度上全面超越现有SOTA，尤其在大视角变化下保持极低的旋转和平移误差，同时生成视频的感知质量依然很高。

### 7. 优点
- **新颖的调节信号**：首次将配对3D点轨迹作为V2V重渲染的显式条件，解决了4D一致性问题。
- **几何可解释性与泛化性**：双视图轨迹调节器中的几何操作完全无参数，模型不记忆特定相机运动，泛化能力强。
- **数据高效构建**：通过时间拼接多相机视频自动提取一对一轨迹对应，无需人工标注。
- **指标提升显著**：相机控制精度大幅提升，尤其在平移误差上降低超过60%，且在其他质量指标上保持领先。
- **模块化设计**：LoRA微调冻结基座模型，训练开销可控，易于适配其他预训练视频生成模型。

### 8. 不足与局限
- **依赖上游跟踪器**：模型性能受制于3D点跟踪器（SpatialTrackerV2）的精度，在跟踪失败（如严重遮挡、无纹理区域）时条件信号可能变差。
- **合成数据训练**：模型完全基于UE5合成数据训练，虽在真实数据（RealEstate10K, MiraData）上表现良好，但其对极端分布外场景（如高度抽象、风格化、复杂动态）的鲁棒性尚不明确。
- **未处理极端遮挡**：虽然时间聚合可应对部分遮挡，但长期完全遮挡的轨迹能否有效恢复仍存疑。
- **计算成本**：推理仍需依赖3D跟踪器，增加了前处理开销，且模型本身需要扩散模型采样，生成效率可能受制。
- **评价局限性**：相机精度依赖SfM提取的位姿与真值比较，在动态场景中SfM可能不稳定，间接影响RotErr/TransErr的可靠性。
