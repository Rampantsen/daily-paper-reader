## 摘要
我们提出WireframeDETR，这是我们提交给结构化语义三维重建(S23DR) 2026挑战赛的方法，该挑战赛要求从多视图COLMAP点云中预测三维建筑线框。我们的方法将DETR风格的集合预测直接应用于三维点云，生成线框作为边坐标对的集合，无需任何中间顶点检测阶段。我们引入了三项技术贡献：(1) 对比去噪训练，可在早期epoch稳定嘈杂的匈牙利匹配；(2) 多尺度编码器，通过学习标量权重聚合最后编码器层的输出；(3) 渐进辅助损失加权，将梯度信号集中在最受益的解码器层上。我们的模型在公开测试集上达到了HSS 0.575 (F1≈0.664, IoU≈0.516)，在清理后的验证拆分上最佳验证HSS为0.534。

## Abstract
We present WireframeDETR, our submission to the Structured Semantic 3D Reconstruction (S23DR) 2026 Challenge, which requires predicting a 3D building wireframe from multi-view COLMAP point clouds. Our method applies DETR-style set prediction directly to 3D point clouds, producing wireframes as sets of edge coordinate pairs without any intermediate vertex detection stage. We introduce three technical contributions: (1) contrastive denoising training that stabilises noisy Hungarian matching in early epochs; (2) a multi-scale encoder that aggregates the last encoder layer outputs via learned scalar weights; and (3) progressive auxiliary loss weighting that concentrates gradient signal on the decoder layers that most benefit from it. Our model achieves a public test HSS of 0.575 (F1~=~0.664, IoU~=~0.516) and a best validation HSS of 0.534 on the cleaned val split.

## 2. 方法论

核心思路是将线框视为一个“边的集合”，每个边被编码为 6D 向量 `(x1, y1, z1, x2, y2, z2)`，并用 Transformer 解码器直接输出固定数量的预测边，再通过匈牙利匹配与真值边对齐。

### 2.1 整体架构
- **点云预处理**：利用 Gestalt 语义标签为顶点/边类点分配更高采样权重，采样 `N=4096`（训练）或 `7168`（推理）个点，仅保留 RGB 颜色（归一化），坐标归一化到 `[0,1]³`。
- **编码器**：线性投影（3→384 维） + 3D 正弦位置编码 → 4 层 Transformer 编码器。
- **解码器**：5 层 Transformer 解码器，使用 `Q=128` 个可学习边查询。
- **输出头**：分类头（边是否存在） + 6D 坐标回归头。
- **训练匹配**：匈牙利算法以联合成本（L1 距离 + 负分类概率）将预测边与真值边配对。

### 2.2 三项技术贡献
1. **对比去噪训练（CDN）**  
   借鉴 DN‑DETR，在每个训练步中为每一条真值边生成正噪声查询（小噪声，`λ+=0.4`）和负噪声查询（大噪声，`λ-=0.8`），分别监督为对应真值边或背景。通过注意力掩码隔离去噪查询与学习查询，保证梯度信号独立。该机制避开了早期训练中匈牙利匹配不稳定的问题，使梯度从第 0 个 epoch 起就得到有效传播。

2. **多尺度编码器记忆**  
   不只使用编码器最后一层输出，而是对最后 `K=3` 层输出进行可学习的加权求和：  
   `M = Σ_k (softmax(w_k) * H_{L–K+k})`。权重 `w_k` 初始化为 0，额外仅增加 3 个参数，使解码器同时获得局部几何和抽象结构特征。

3. **渐进辅助损失加权**  
   对解码器中间层的辅助损失施加线性递增的权重：第 `i` 个辅助输出的权重为 `0.5 + 0.5*(i+1)/N_aux`，范围从 0.6 增至 1.0，将更多梯度信号导向预测质量更高的后层。

### 2.3 后处理与推理
- 保留分类得分 >0.9 的预测边，将边端点去重：合并距离 <0.5m 的顶点，并以入射边方向计算新位置。
- 可选的旋转增强推理（0°, 90°, 180°, 270° 四种 Y 轴旋转），融合各方向预测后去重，但默认不开启。

## 6. 主要结论与发现

- 通过 DETR 式集合预测直接回归 3D 边坐标，**无需顶点检测中间阶段**，即可高效重建建筑线框。
- 对比去噪训练能可靠解决早期匈牙利匹配不稳的问题，使得 CDN 坐标损失从 2.07 稳步降至 1.25，分类损失同步下降。
- 结合多尺度编码器记忆和渐进辅助损失加权，最终模型在 S23DR 2026 公开排行榜上取得了**HSS 0.575**，大幅超越已有基线。

## 7. 优点

- **端到端与简洁性**：避免了传统多阶段方法中的级联误差和后处理 NMS，天然支持变基数的线框输出。
- **训练稳定性**：对比去噪让模型从第 0 个 epoch 起获得稳定的梯度信号，显著缓解了 DETR 早期训练困难。
- **轻量设计**：多尺度聚合仅增加 3 个学习参数，渐进损失加权无需额外计算开销。
- **实践指导价值**：清晰记录了被放弃的路径（Perceiver 微调、两阶段 PointNet）及失败原因，为后续研究者提供参考。

## 8. 不足与局限

- **计算效率**：训练速度约为 Perceiver 基线的 2 倍，且 CDN 可变序列长度阻碍 `torch.compile` 优化，带来额外开销。
- **推理耗时**：若启用 4 次旋转的 TTA，推理时间乘以 4，不利于实时或大规模部署。
- **特征利用有限**：仅使用了 3 通道 RGB，未融合 Gestalt 语义标签或深度估计等直接输入特征（仅用于采样），可能限制了性能上限。
- **实验完整性**：缺少对关键组件（如 CDN、多尺度编码器、损失加权）的独立消融实验，难以量化每项贡献的大小；也未探索查询数量、采样点数对性能的影响。
- **泛化边界**：数据集限定于建筑屋顶，模型的泛化至其他类型建筑或不同点云密度/噪声水平的场景未经验证。
