## 摘要
我们提出了MoVerse，一种实时视频世界模型，能够从单张窄视场图像创建可交互导航的场景。这一设置具有挑战性，因为输入只观察到环境的一小部分，而交互式漫游需要完整的周围世界、持久的几何结构、可控的摄像头运动以及时间上连贯的高保真观测。MoVerse通过将世界构建与观测渲染分离来解决这一问题。它首先利用拓扑感知扩散将输入扩展为重力对齐的360°全景图，在三维推理之前填补缺失的视野。然后使用全景几何感知残差预测将全景图提升为持久的3D高斯支架，产生密集且可直接渲染的空间记忆。最后，一个高斯条件视频渲染器将沿用户指定相机轨迹的支架渲染转化为照片级真实的视频。为了使该渲染器适用于交互，我们训练了一个双向扩散教师模型来进行高质量的条件渲染，并将其蒸馏为一个因果自回归学生模型以实现有界延迟流式传输。这种设计结合了显式三维表示的可控性和长程一致性以及生成式视频模型的感知质量。MoVerse在单个NVIDIA RTX 4090 GPU上支持8 FPS的实时场景漫游，展示了一条通往具有交互式视频输出的单图像世界创建的实用路径。

## Abstract
We present MoVerse, a real-time video world model that creates an interactively navigable scene from a single narrow-field-of-view image. This setting is challenging because the input observes only a small fraction of the environment, while interactive roaming requires a complete surrounding world, persistent geometry, controllable camera motion, and temporally coherent high-fidelity observations. MoVerse addresses this problem by separating world construction from observation rendering. It first expands the input into a gravity-aligned 360$^\circ$ panorama with topology-aware diffusion, closing the missing field of view before 3D reasoning. It then lifts the panorama into a persistent 3D Gaussian scaffold using panoramic geometry-aware residual prediction, yielding a dense and directly renderable spatial memory. Finally, a Gaussian-conditioned video renderer translates scaffold renderings along user-specified camera trajectories into photorealistic video. To make this renderer practical for interaction, we train a bidirectional diffusion teacher for high-quality conditional rendering and distill it into a causal autoregressive student for bounded-latency streaming. This design combines the controllability and long-range consistency of explicit 3D representations with the perceptual quality of generative video models. MoVerse supports real-time scene roaming at 8~FPS on a single NVIDIA RTX~4090 GPU, demonstrating a practical path toward single-image world creation with interactive video output.

## 2. 方法论

### 核心思想
分三个阶段逐步完成从窄视场图像到可交互视频流的转换，各阶段职责清晰，既可独立优化，又可模块化替换。

### 关键技术细节

**Stage I – 全景生成（Panoramic Generation）**
- **目标**：将无姿态的单张NFOV图像 \(I\) 扩展为重力对齐、水平闭合的360°等距矩形投影（ERP）全景图 \(P\)。
- **重力对齐与可微分自动调平**：
    - 先预测稠密二维对应场 \(P_{\text{dense}}\)，通过可微软argmin求解器拟合刚性球面旋转 \(\hat{R}\)，去除俯仰和翻滚，使水平线对齐ERP赤道。
    - 采用球面网格采样进行warp，利用稠密对应场监督 \(L_{\text{flow}}\) 但通过刚性瓶颈避免非刚性扭曲。
- **拓扑感知全景补全**：
    - 使用带掩码的潜在扩散模型，将输入图像编码后作为已知区域上下文。
    - 圆型潜在编码：潜在自编码器的水平卷积填充具备循环属性，避免ERP边界虚假接缝。
    - 移位等变扩散：引入水平循环移位操作，并要求基础输出与移位输出满足 \( \operatorname{Roll}_{\delta}(\hat{\epsilon}_{\text{base}}) = \hat{\epsilon}_{\text{shifted}} \)，通过 \(L_{\text{shift}}\) 正则化，迫使模型消除接缝。
- **训练数据**：自行构建的 Horizon360 数据集，将原始全景图通过旋转使地平线与赤道对齐，采样多样视角（不同偏航、俯仰、翻滚、视场）并始终将观测视角置于全景中心（\(P_{\text{centered}} = \operatorname{Roll}(P_{\text{ERP}}, -\psi_{\text{input}})\)）。
- **损失函数**：\(L_{\text{stage1}} = L_{\text{LDM}} + \lambda_{\text{shift}}L_{\text{shift}} + \lambda_{\text{flow}}L_{\text{flow}}\)。

**Stage II – 高斯场景生成（Gaussian Scene Generation）**
- **目标**：将全景图 \(P\) 转换为可直接渲染的持久3D高斯支架 \(G\)。
- **全景高斯初始化**：
    - 给定估计的全景深度图 \(D\)，在步进ERP网格上初始化 \(K\) 个高斯原语。
    - 位置通过球面反投影得到 \(\mu_k = D_k d(\theta_k, \phi_k)\)。
    - 根据纬度校正初始尺度：\(s_k \propto D_k \cos \phi_k\)（极区退化加以裁剪）。
- **残差预测**：
    - 从全景图和逆深度输入提取多尺度特征，使用 DPT 风格头解码为几何和纹理特征 \(f_k\)。
    - 轻量残差头 \(h_\psi\) 预测在角度-逆深度空间中的残差：
      \[
      \theta'_k = \theta_k + \lambda_{xy}\Delta\theta_k,\quad \phi'_k = \operatorname{clip}(\phi_k + \lambda_{xy}\Delta\phi_k,\; -\pi/2+\epsilon,\; \pi/2-\epsilon),\quad D'_k = 1/\big(\operatorname{softplus}(\rho_k + \lambda_z\Delta z_k) + \epsilon\big)
      \]
    - 通过球面投影得到更新后的中心 \(\mu_k\)；其它属性（尺度、旋转、颜色、不透明度）也以残差方式更新。
- **训练目标**：
    - 目标视角渲染损失：\(L_{\text{tgt}} = \lambda_1\|R(G)-I_{\text{tgt}}\|_1 + \lambda_{\text{lpips}}\text{LPIPS} + \lambda_{\text{ssim}}(1-\text{SSIM})\)。
    - 源视角重建损失、逆深度损失 \(L_{\text{depth}}\) 和位置残差正则 \(L_{\Delta}\)。
    - 总损失：\(L_{\text{GS}} = L_{\text{tgt}} + \lambda_{\text{src}}L_{\text{src}} + \lambda_{\text{depth}}L_{\text{depth}} + \lambda_{\Delta}L_{\Delta}\)。
- **训练数据**：HM3D 数据集。

**Stage III – 高斯条件视频渲染（Gaussian-Conditioned Video Rendering）**
- **目标**：沿用户相机轨迹将支架渲染的RGB序列 \(\hat{V}_{1:T}\) 提升为高保真输出视频 \(V_{1:T}\)，并实现实时流式。
- **双向条件教师**：
    - 基于 Wan2.1-T2V-1.3B 扩散 Transformer 构建，输入为带噪目标潜在 \(z_t\) 与条件潜在 \(c\)（来自支架渲染的视频编码）的拼接 \(\tilde{z}=[z_t \| c]\)。
    - 使用共享的旋转位置编码（RoPE），使条件与目标在坐标空间中精确对齐。
    - 以流匹配目标训练：\(L_{\text{teacher}} = \mathbb{E}\| v_\theta(\tilde{z}, \tau, t) - (\epsilon - z) \|_2^2\)。
- **因果自回归学生与蒸馏**：
    - 将双向教师蒸馏为因果学生：每次生成一个潜在块（对应4个像素帧），仅看见历史缓存。
    - 蒸馏采用 Self Forcing 初始化和 RAVEN 风格的分布匹配，使学生在自回归环境下学习自身上下文编码。
    - 最终在线推理时使用 MemRoPE 局部 KV 缓存（1个 sink 帧、1个长期EMA令牌、3帧滑动窗口）和 TAEHV 轻量解码器。
- **训练数据混合**：渐进式三族数据——
    1. 真实视频 + 深度投影（训练修复由相机平移引起的新区域暴露）；
    2. 稀疏关键帧高斯支架渲染（模拟泼溅伪影、浮动物等）；
    3. 全流程全景→高斯支架渲染（最贴近部署条件，来自 LayerPano3D、Matterport3D、Polyhaven HDRI 等）。
- **实时推理**：每步1个潜在块，2步去噪，结合上述缓存策略，在单卡RTX 4090上达到8 FPS。

## 7. 优点

- **清晰的三阶段分离设计**：问题分解合理，每阶段目标明确，降低耦合度，利于后续优化。

- **实用的蒸馏方案**：通过双向→因果蒸馏及分布匹配，使高质量条件视频模型能转化为实时流式推理，同时使用共享RoPE增强条件对齐。
