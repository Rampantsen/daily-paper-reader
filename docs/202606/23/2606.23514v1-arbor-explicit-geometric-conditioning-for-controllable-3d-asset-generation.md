## 摘要
基于文本和图像条件的三维模型现在能够生成令人信服的资产，但它们对物体应占据或避让的空间仍缺乏直接控制。在创作过程中，这种空间意图往往在生成开始前就已明确。椅子应适配一个座位包络，道具应为运动留出间隙，或部件应露出接触面。提示词和图像视图难以承载这类约束，因此需要一种显式的控制界面。我们提出 Arbor，一个可训练的附加模块，用于文本条件引导的隐式三维生成。Arbor 引入约束网格作为原生的三维控制界面。该界面使用应存在几何体的外壳区域、应保持空闲的避让区域以及物体应接触的接触区域。与补全或整体物体骨架控制不同，这些网格并非目标证据，而是局部类型化要求，并可包含不应出现曲面的区域。Arbor 将约束网格转换为令牌，并在冻结的去噪器内部学习一个路由式附加结构，从而将这一信号保留为几何形式。因此，每个隐式区域都能接收到与其空间位置相关的部分约束信息。我们在外壳、避让和接触约束下，通过自动化和艺术家策划的控制基准评估 Arbor，并将指标趋势与用户偏好研究进行比较。即使没有专门的合规损失，Arbor 仍能在固定约束下改善约束服从性，同时保持物体质量和变化性。

## Abstract
Text and image conditioned 3D models now generate convincing assets, but they still offer little direct control over the space an object should occupy or avoid. In authoring, this spatial intent is often known before generation starts. A chair should fit a seating envelope, a prop should leave clearance for motion, or a part should expose a contact surface. Prompts and image views are poor carriers for such constraints, requiring the need for an explicit control interface.   We present Arbor, a trainable attachment for text conditioned latent 3D generation. Arbor introduces constraint meshes as a native 3D control interface. The interface uses hull regions where geometry should exist, avoidance regions that should remain empty, and touch regions the object should contact. Unlike completion or whole object scaffold control, these meshes are not target evidence. They are local typed requirements and can include regions where no surface should appear. Arbor keeps this signal as geometry by converting constraint meshes into tokens and learning a routed attachment inside a frozen denoiser. Each latent region can therefore receive the part of the constraint that matters for its spatial location.   We evaluate Arbor on automatic and artist curated control benchmarks with hull, avoidance, and touch constraints, and compare the metric trends to a user preference study. Even without dedicated compliance losses, Arbor improves constraint obedience while preserving object quality and variation under fixed constraints.

### 2. 论文提出的方法论

Arbor 的核心是将用户指定的约束网格转化为紧凑的几何令牌，并通过一个路由式残差分支注入到冻结的潜在去噪器中，使生成过程服从空间约束。

*   **核心思想：**
    *   **约束网格作为原生控制界面**：用户通过直观的三维网格定义三种类型化空间要求。
    *   **约束编码与对齐**：利用一个已冻结的、专为表面属性设计的几何编码器，将约束网格及其类型信号转换为与生成器潜在空间兼容的令牌。
    *   **路由式适配器注入**：在生成器的每个 Transformer 块中，通过一个几何路由器为不同空间区域的潜在查询提供最相关的局部约束信息，并以残差方式注入，实现“空间知情”的生成。

*   **关键技术细节与流程：**
    *   **约束定义**：
        *   `外壳约束`: 标记生成几何体**应该存在**的区域。
        *   `避让约束`: 标记生成几何体**不应存在**的区域，强调“空”的空间。
        *   `接触约束`: 标记资产**应该接触**的表面，但不过分延伸。
    *   **约束编码**：
        1.  **输入融合**：将多个约束网格融合为一个统一网格 `Cmesh`，并为其面片分配类型信号通道（`Csignal`，二进制标志）。
        2.  **OVoxel化**：使用 TRELLIS.2 的方法将约束网格体素化为形状场和属性场。
        3.  **冻结编码器**：利用 TRELLIS.2 冻结的形状编码器处理形状场，得到 `cshape`；同时，复用其原本用于材质的属性编码器处理法向量场和约束信号场，得到 `csignal`。
        4.  **令牌集成**：将 `cshape` 和 `csignal` 拼接成几何记忆令牌 `cgeo`，并通过可学习的线性投影和三维位置嵌入，使其与主生成器 TRELLIS.1 的潜在空间对齐。
    *   **路由式适配器控制**：
        1.  **模型主干**：基于 TRELLIS.1 的稀疏结构去噪 Transformer。每个 Transformer 块包含自注意力、文本交叉注意力和前馈网络。
        2.  **适配器插入**：在文本交叉注意力之后、前馈网络之前，插入一个几何接地分支，以残差方式更新潜在状态 (`z`)。新状态更新公式为 `z = FFN(z_text + W_geo * Δz_geo)`，其中 `Δz_geo` 是几何更新量，`W_geo` 初始化为零。
        3.  **几何路由器**：
            *   **查询分组**：将 `16³` 的潜在查询空间划分为 64 个 `4×4×4` 的局部组。
            *   **局部路由**：对于每个查询组 `g`，通过其中心点和八个角点作为路由锚点，计算所有几何令牌到最近锚点的距离，然后**选择 K=2048 个距离最近的令牌**作为该组的局部上下文 `c(g)local`。这使得每个区域只关注其附近的约束信息。
            *   **全局总结**：使用 96 个可学习的总结令牌 `Sglobal`，通过交叉注意力机制对所有几何令牌进行总结，得到全局上下文 `cglobal`，以提供物体级的广度信息（如整体尺寸、接触方向）。
            *   **上下文组合**：最终每个查询组的上下文为局部和全局令牌的拼接：`c(g)ctx = [c(g)local; cglobal]`。
        4.  **几何更新**：局部组内的查询状态 `z_text` 作为 Query，对该组的上下文 `c(g)ctx` 进行交叉注意力计算，得到几何更新量 `Δz_geo`。
    *   **训练策略**：
        *   **冻结骨干**：TRELLIS.1 和 TRELLIS.2 编码器的所有权重均被冻结。
        *   **可训练模块**：仅训练几何投影层、位置嵌入、全局总结模块和交叉注意力适配器。
        *   **目标函数**：直接使用 TRELLIS.1 原始的流匹配损失，**不引入任何显式的约束合规性损失**。
        *   **条件丢弃**：训练时对文本和几何条件独立地进行丢弃，以支持无分类器引导。

### 6. 论文的主要结论与发现

*   **优越的控制服从性**：Arbor 在保持生成物体质量和变化性的同时，相比于原始骨干网、采样时引导方法和训练式基线，显著提高了对“外壳”、“避让”和“接触”几何约束的服从度。
*   **控制信号的有效注入**：学习式的路由适配器是注入几何约束的最佳方式，其效果显著优于仅在采样时施加梯度引导。
*   **无需合规损失即可工作**：仅通过原始的生成目标训练适配器，Arbor 就能学会遵循约束，而添加显式的合规损失可能导致模型倾向于“过度拟合”约束，损害物体本身的可信度。
*   **维持生成多样性**：在固定约束下，Arbor 能够生成多样化的资产，其变化性远超图像锚定的重建类方法，兼顾了控制的精确性和创意空间。
*   **用户偏好验证**：用户偏好研究结果与自动指标趋势高度一致，Arbor 系列方法获得了最高的胜率，证明了其方法的实际优越性。

### 7. 优点

*   **问题定义的创新性**：首次在文本条件三维生成中引入了明确的、类型化的空间约束（外壳、避让、接触），特别是“避让”和“接触”的提出，填补了“空”和“附着”控制的空白。
*   **方法论的技术精巧性**：
    *   **编码器的复用**：巧妙地复用了为材质设计的 TRELLIS.2 属性编码器来编码几何控制信号，避免了从零开始设计编码器。
    *   **几何路由器的设计**：通过局部与全局结合的路由机制，高效且精准地将高密度的几何令牌信息传递给潜在空间中对应的区域，解决了尺度、稀疏性和成本问题。
    *   **即插即用性**：作为冻结骨干网的适配器，保留了原始生成器的强大先验，同时为控制提供了专用的模块，训练高效。
*   **实验设计的严谨性**：
    *   **双基准评估**：结合自动生成与艺术家手工制作的评估集，兼顾了规模和泛化性测试。
    *   **公平对比意识**：明确区分了不同任务设定（纯文本条件 vs. 多模态条件）的方法，并分轨道比较。
    *   **“Ctrl Score”指标**：提出的调和平均数指标能综合反映控制能力与生成质量的平衡，避免了单一指标的片面性。

### 8. 不足与局限

*   **缺乏语义功能控制**：约束区域目前只携带几何形状和类型信号，但不携带完整的语义功能信息。例如，指定了一个“座位”区域，但无法保证生成器在提示词冲突时会将其用作座位。论文中的语义变体（Arbor Semantics）尚未表现出超过纯几何路径的性能。
*   **局限于生成早期阶段**：Arbor 仅在 TRELLIS 框架的“稀疏结构生成”阶段起作用，不直接控制后续的“潜在细化”阶段（如表面细节、材质属性生成），可能导致精细几何细节与粗结构约束不完全对齐。
*   **约束与文本先验的潜在冲突**：当约束与文本提示强烈冲突（如极小外壳、激进避让）时，模型可能需要在“服从提示”和“服从约束”之间妥协，导致某一方明显失败。
*   **基线对齐的固有困难**：由于问题设定（纯文本-几何 vs. 图-几何）不同，与某些方法（如重建模型）的比较不在完全公平的起跑线上，只能通过分赛道实验来部分缓解。
*   **数据生成依赖**：训练高度依赖于一个复杂的程序化约束生成系统，该系统的设计和质量直接影响模型的学习效果。
