## 摘要
我们提出EvObj，用于无监督三维实例分割，弥合合成预训练数据与真实点云之间的几何领域差距。当前方法在将对象先验从合成数据集（如ShapeNet）迁移到真实扫描数据（如ScanNet）时面临结构差异，尤其是形态变化和遮挡伪影。为此，EvObj整合了两个创新模块：（1）对象辨别模块，动态细化对象候选，使对象先验持续适应目标域；（2）对象补全模块，在发现对象后重建部分几何形状。我们在真实世界和合成数据集上进行了广泛实验，表明其在所有基线方法上均表现出优越的三维对象分割性能，并达到了最先进水平。

## Abstract
We introduce EvObj for unsupervised 3D instance segmentation that bridges the geometric domain gap between synthetic pretraining data and real-world point clouds. Current methods suffer from structural discrepancies when transferring object priors from synthetic datasets (e.g., ShapeNet) to real scans (e.g., ScanNet), particularly due to morphological variations and occlusion artifacts. To address this, EvObj integrates two innovative modules: (1) An object discerning module that dynamically refines object candidates, enabling continuous adaptation of object priors to target domains; and (2) An object completion module that reconstructs partial geometries after discovering objects. We conduct extensive experiments on both real-world and synthetic datasets, demonstrating superior 3D object segmentation performance over all baselines while achieving state-of-the-art results.

### 二、方法论
#### 核心思想
基于GrabS的两阶段框架（预训练对象中心网络+策略网络），引入两个可训练模块：**对象辨别与演化模块**和**对象补全模块**，在策略网络训练过程中动态适应目标域，并修复残缺几何。

#### 关键技术细节
1. **对象辨别与演化模块（f<sub>dis_evo</sub>）**
   - **结构**：逐点二值分割网络（采用SparseUNet），输入策略网络裁剪的点集 P<sub>c</sub>，输出前景/背景掩码 M<sub>c</sub>。
   - **阶段一（预训练）**：在合成数据上训练，用随机平面模拟墙面/地面作为噪声，学会从混合点云中辨别单个物体。
   - **阶段二（演化）**：在策略网络训练期间，每隔 T 个epoch，利用**最新累积的高对象性评分候选**作为伪标签，对 f<sub>dis_evo</sub> 进行微调，微调后抛弃伪标签。这一策略使网络持续适应目标域中不断出现的物体形态变化。

2. **对象补全模块（f<sub>comp</sub>）**
   - **结构**：AdaPoinTr，预训练于ShapeNet，输入部分点云，预测完整形状 O<sub>c</sub>。
   - **作用**：将辨别模块输出的（通常是残缺的）对象候选补全后再送入对象中心网络评分，避免因遮挡导致评分过低，从而误导策略网络。

3. **整体流程**
   - 预训练阶段：固定对象中心网络 f<sub>obj</sub>，辨别模块预训练，补全模块预训练。
   - 训练阶段：策略网络 f<sub>pol</sub> 控制动态容器裁剪点集 P<sub>c</sub> → f<sub>dis_evo</sub> 预测掩码 → f<sub>comp</sub> 补全 → f<sub>obj</sub> 评分 → 评分作为奖励优化 f<sub>pol</sub>（PPO）→ 高评分候选作为伪标签微调 f<sub>dis_evo</sub>（演化）并最终训练前馈分割网络 f<sub>seg</sub>。
   - 推理阶段：直接用训练好的 f<sub>seg</sub> 对输入场景进行分割。

- **训练开销较大**：总训练时间104小时，比基线GrabS（91小时）更长，虽然推理速度相当。
