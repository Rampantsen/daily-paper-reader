## 摘要
三维语言高斯泼溅（3DLGS）通过语言对齐的视觉特征增强三维高斯泼溅，以实现开放词汇的三维场景理解。核心挑战在于高效地将高维视觉-语言嵌入与数百万个三维高斯关联起来，同时保持基于文本查询的高效特征渲染。现有方法要么直接在高斯上存储密集特征，导致存储成本高、渲染速度慢，要么通过昂贵的逐场景优化和重复特征光栅化来学习紧凑表示。目前尚无方法能同时实现快速三维语义重建、高效存储和快速渲染。我们提出 SCOUP（稀疏码本提升），通过将语言表示学习与三维高斯优化解耦来解决这三个问题。我们完全利用二维图像区域关联的特征来学习基于稀疏码本的表示，而不是直接在三维空间工作，将每个区域与一组稀疏的码本系数关联起来。然后，利用高斯到像素的关联，通过加权稀疏聚合将这些系数提升到三维高斯，每个高斯在不同视图上累积码本原子的系数。接着，Top-$K$ 过滤提取每个高斯最显著的多视图系数，从而实现高效存储和快速渲染。与现有最优方法相比，我们的方法在训练速度上实现了高达 $400\times$ 的加速，同时训练内存效率提升 $3\times$。在多个基准测试中，SCOUP 在开放词汇查询准确率上达到或超越了现有方法。

## Abstract
3D Language Gaussian Splatting (3DLGS) augments 3D Gaussian Splatting with language-aligned visual features for open-vocabulary 3D scene understanding. A core challenge is efficiently associating high-dimensional vision-language embeddings with millions of 3D Gaussians while preserving efficient feature rendering for text-based querying. Existing methods either store dense features directly on Gaussians, causing high storage costs and slow rendering, or learn compact representations through expensive per-scene optimization with repeated feature rasterization. No existing method simultaneously achieves fast 3D semantic reconstruction, efficient storage, and fast rendering. We propose SCOUP (Sparse COde UPlifting), which addresses all three by decoupling language representation learning from 3D Gaussian optimization. Rather than working directly in 3D, we learn sparse codebook-based representations entirely using features associated with 2D image regions, associating each region with a sparse set of codebook coefficients. We then uplift these coefficients to 3D Gaussians with our weighted sparse aggregation using Gaussian-to-pixel associations, where each Gaussian accumulates coefficients over codebook atoms across views. Top-$K$ filtering then extracts the most dominant multi-view coefficients per Gaussian, enabling efficient storage and fast rendering. Our method achieves up to $400\times$ training speedup while being $3\times$ more memory efficient during training compared to the state-of-the-art in rendering speed. Across multiple benchmarks, SCOUP matches or outperforms existing methods in open-vocabulary querying accuracy.

## 1. 核心问题与整体含义
在三维语言高斯泼溅（3DLGS）任务中，核心挑战在于如何高效地将高维视觉-语言嵌入（如CLIP特征）关联到三维场景的数百万个高斯体上，同时保证基于文本查询的渲染速度。现有方法存在两大矛盾：
- **存储密集特征的方法**（如直接特征提升）导致巨大的存储开销与缓慢的渲染。
- **优化压缩表示的方法**（如LangSplat系列）需要昂贵的逐场景优化与重复的特征光栅化，训练极慢，且容易丢失小物体语义。
目前没有方法能**同时**实现快速的3D语义重建、高效的存储与快速的渲染。SCOUP旨在填补这一空白，提出一种高效且精准的3DLGS框架。

## 2. 方法论：核心思想与关键技术
SCOUP提出 **“解耦”** 策略：将语言表示学习与3D高斯优化分离，整个过程分为三步。
- **2D稀疏系数编码**：利用SAM分割出图像区域，用CLIP提取特征后，在**二维图像空间**直接学习一个共享的全局码本（codebook）和每个区域的稀疏系数（最多K个非零值）。通过归一化Soft Top-K约束与余弦重建损失优化，将高维特征无损压缩为稀疏系数图。这避免了在百万级高斯上的复杂优化，且小物体与大物体在码本学习中贡献更均衡。
- **稀疏系数提升**：利用预训练3DGS提供的高斯-像素关联权重（$\alpha$混合权重），将二维稀疏系数**无优化地加权聚合**到每个三维高斯上。每个高斯从多视图积累码本原子的系数，这个过程仅操作非零项，复杂度与特征维度D无关，只与稀疏度K有关，实现了$D/K$倍加速。
- **Top-K过滤与渲染**：对聚合后的系数向量执行Top-K选择，保留每个高斯最显著的K个系数。这相当于一次多视图语义投票去噪：高权重系数代表跨视图共识，低权重系数被过滤。最终每个高斯仅存储K个系数索引与值，渲染时只需稀疏聚合和与码本的矩阵乘法，极大提升存储与渲染效率。

关键公式可概括为：
- 二维稀疏编码：$F_v(p) \approx W_v(p) C$，满足 $\|W_v(p)\|_0 \le K$，非负且和为1。
- 稀疏提升：$w_i = \frac{1}{Z_i}\sum_{(v,p)\in S_i} e_i(v,p) W_v(p)$，然后 $\hat{w}_i = \text{Top}_K(w_i) / \|\text{Top}_K(w_i)\|_1$。
- 渲染恢复：$\hat{F}_v(p) = (\sum_i \hat{w}_i e_i(v,p)) C$。

## 4. 资源与算力
- **硬件环境**：所有实验在单块 **NVIDIA RTX A6000 GPU**（48 GB 显存）上运行。
- **训练开销**：SCOUP的3D语义重建时间极短，在LERF上平均约0.5分钟，Mip-NeRF360上约1.4分钟；对比LangSplatV2需560~680分钟（约9~11小时）。2D稀疏编码需训练4000代（使用Adam），整体训练内存峰值大幅降低（例如在Mip-NeRF360上SCOUP仅需10.4 GB，LangSplatV2需37.7 GB）。

## 6. 主要结论与发现
- SCOUP首次在3DLGS中同时实现快速训练（**400+倍加速**）、低存储（每高斯仅存4个系数）与高速渲染（29 FPS），且精度不降反升。
- 二维稀疏编码是更有效的场景语言表示学习方式，比三维联合优化更快、更省内存，且通过Top-K过滤的跨视图投票机制，**提升了多视图一致性，可作为去噪手段直接提升分割和定位精度**（mIoU提升3.9%）。
- 在LERF-OVS上，SCOUP整体mIoU达64.4%，领先LangSplatV2 +4.5%；在其他基准上与最优方法持平或更具竞争力，同时效率优势巨大。

## 7. 优点与亮点
- **方法论创新**：首次将二维稀疏编码与无优化提升结合，巧妙绕过三维联合优化的沉重负担。稀疏提升和Top-K过滤设计简洁而高效。
- **系统效率突出**：将语义重建从小时级降至分钟以内，显存占用极低，极大降低实际部署门槛，对实时应用友好。
- **实验扎实**：横向对比全面，纵向消融细致，公平重复测量基线，分析深入（如熵值解释Top-K作用），可复现性强。

## 8. 不足与局限
- **前置依赖重**：仍需庞大的视觉基础模型（SAM-H, CLIP）进行预处理，这部分耗时未计入3D重建时间，实际端到端流水线仍有瓶颈。
- **鲁棒性受限**：依赖3DGS几何先验，对透明/低不透明度物体的特征提升可能不准确（论文Figure 6予以说明）。

- **模型容量与泛化**：码本和稀疏度需针对场景设定，当前最优参数（L=64, K=4）与LangSplatV2一致，但未探讨跨场景共享码本的可行性及更大规模泛化。
