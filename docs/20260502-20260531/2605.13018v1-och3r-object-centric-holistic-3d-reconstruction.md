## 摘要
以对象为中心的场景理解是计算机视觉中的一个基本挑战。现有方法通常依赖多阶段流程，首先应用预训练的分割器提取单个对象，然后进行逐对象的三维重建。这类方法计算开销大，容易受到分割错误的影响，且难以适应场景复杂度的增加。我们提出了OCH3R，一个从单张RGB图像进行以对象为中心的整体三维重建的统一框架。OCH3R只需一次前向传递即可同时预测所有对象实例的6D位姿和精细的三维重建。其核心是一个Transformer架构，预测每个像素的属性，包括基于CLIP的类别嵌入、度量深度、归一化对象坐标（NOCS）以及表示每个对象的固定数量的三维高斯分布。为了监督这些高斯重建，我们利用预测的6D位姿将其变换到规范空间，并与预渲染的规范真值对齐，从而避免了昂贵的逐图像高斯标签生成。在标准室内基准测试中，OCH3R在单目深度估计、开放词汇语义分割以及仅RGB的类别级6D位姿估计方面均达到了最先进的性能，同时生成了高保真、可编辑的逐对象重建结果。关键的是，推理是完全前馈的，并且其复杂度与对象数量无关，在杂乱场景中比传统多阶段流程快了数个数量级。

## Abstract
Object-centric scene understanding is a fundamental challenge in computer vision. Existing approaches often rely on multi-stage pipelines that first apply pre-trained segmentors to extract individual objects, followed by per-object 3D reconstruction. Such methods are computationally expensive, fragile to segmentation errors, and scale poorly with scene complexity. We introduce OCH3R, a unified framework for Object-Centric Holistic 3D Reconstruction from a single RGB image. OCH3R performs one forward pass to simultaneously predict all object instances with their 6D poses and detailed 3D reconstructions. The key idea is a transformer architecture that predicts per-pixel attributes, including CLIP-based category embeddings, metric depth, normalized object coordinates (NOCS), and a fixed number of 3D Gaussians representing each object. To supervise these Gaussian reconstructions, we transform them into canonical space using the predicted 6D poses and align them with pre-rendered canonical ground truth, avoiding costly per-image Gaussian label generation. On standard indoor benchmarks, OCH3R achieves state-of-the-art performance across monocular depth estimation, open-vocabulary semantic segmentation, and RGB-only category-level 6D pose estimation, while producing high-fidelity, editable per-object reconstructions. Crucially, inference is fully feed-forward and scales independently of the number of objects, offering orders-of-magnitude speedups over conventional multi-stage pipelines in cluttered scenes.

## 2. 方法论
### 2.1 核心思想
- 使用一个 **48 层 Transformer** 对每个像素同时预测多种属性：CLIP 空间语义嵌入、度量深度、归一化物体坐标（NOCS）、以及每个像素固定的 k=2 个 3D 高斯。
- 推理时通过**聚类获得实例**，利用 NOCS 与 3D 点对应关系**估计 SIM(3) 位姿**，再将每物体的高斯变换到**规范空间**，组装成可交互的物体云。
- 训练时采用**规范空间监督（CSS）**：将预测的高斯经真值位姿变换到规范帧，与该物体预渲染的多视角真值图像对齐，避免生成逐图像的昂贵高斯标签，并实现非模态补全。

### 2.2 关键公式与流程
- **像素级预测**：对于像素 (u,v)，网络输出
  \[
  \Phi(I)_{u,v}=(\hat{e}_{u,v},\ \hat{d}_{u,v},\ \hat{c}_{u,v},\ \hat{G}_{u,v})
  \]
  其中 \(\hat{e}\) 为 512 维 CLIP 嵌入，\(\hat{d}\) 为度量深度，\(\hat{c}\) 为 NOCS 坐标，\(\hat{G}\) 包含 k 个高斯（均值、协方差、透明度、球谐系数）。
- **实例发现**：利用全连接条件随机场（CRF）在余弦相似度空间聚类，形成各物体的像素掩码 \(\hat{P}_j\)。
- **位姿估计**：通过 RANSAC+Umeyama 最小化下式解出各物体 SIM(3) 变换 \(\hat{\Pi}_j\)
  \[
  \hat{\Pi}_j = \arg\min_{\Pi} \sum_{(u,v)\in\hat{P}_j} \|\hat{p}_{u,v} - \Pi(\hat{c}_{u,v})\|^2
  \]
  \(\hat{p}_{u,v}\) 由深度反投影得到。
- **高斯构建与训练**：将相机空间高斯均值 μ 经 \(\hat{\Pi}^{-1}_j\) 变换至规范空间，在规范空间内渲染多视角图像 \(\hat{I}_n\)，并与预渲染真值 \(I_n^{\mathrm{gt}}\) 计算 CSS 损失：
  \[
  L_{\mathrm{CSS}} = \sum_{n=1}^{N} \big(\|I_n^{\mathrm{gt}} - \hat{I}_n\|_1 + \lambda_{\mathrm{SSIM}}(1 - \mathrm{SSIM}(I_n^{\mathrm{gt}}, \hat{I}_n))\big)
  \]
- **架构细节**：基于 DINOv2 骨干，后接 48 层 ViT 编码器及四个 DPT 风格解码头（语义、深度、NOCS、高斯），并附加一个相机头预测视场角。高斯头分为几何（预测相机空间偏移）和外观/形状（预测规范空间参数）两部分。

### 2.3 训练策略
- 联合优化深度（逆深度 L2）、语义（交叉熵）、NOCS（分类+偏移回归）、高斯（CSS）和相机 FOV（Huber）损失，采用同方差不确定性加权。
- 使用大规模整合数据集（PACE、Omni6DPose、GSO、HyperSim），共约 92 万张训练图像。

## 6. 主要结论与发现
- OCH3R 在所有评测的物体中心三维重建基准上，以显著优势超越多阶段/扩散式基线，同时获得**数量级的推理加速**（约 0.7 秒/图 vs 数十分钟甚至数十分钟）。
- 其统一的逐像素预测不仅支持高质量重建，还在**单目深度估计、开放词汇语义分割、类别级 6D 位姿估计**上达到或接近最优零样本性能，验证了多任务联合学习的相互促进。
- 规范空间监督配合相机空间偏移，是训练非模态完整高斯表示的关键，无需逐图像真值。

## 7. 优点
- **统一端到端**：将分割、深度、位姿、重建融合为一次前馈，消除多阶段累积误差，推理速度极快且几乎不随物体数量增长。
- **规范空间监督**：巧妙避开了生成逐像素/逐图像高斯标签的高昂成本，同时实现遮挡区域的非模态补全。
- **多任务互益**：语义、几何、位姿任务共享骨架，相互提升，较强的泛化能力。
- **架构灵活**：方法可与不同密集预测骨架（ViT、U-Net）结合，随模型容量增大性能平滑提升。

## 8. 不足与局限
- **反射/透明表面**：镜面、玻璃等物体常导致深度和 NOCS 估计错误，重建不稳定。
- **深度‑高斯耦合偏差**：高斯初始位置依赖于预测深度，当深度出现系统性误差时（如物体薄边界），会引起高斯偏移，可能出现轻微对齐问题。
- **背面模糊**：由于是单次前馈而非迭代式生成（如扩散模型），不常见的背面或复杂拓扑结构可能会出现过度平滑、细节缺失。
- **实验覆盖**：虽然测试集丰富，但主要集中于桌面/室内物体，对户外大规模场景或其他尺度的泛化性未验证；此外与某些需要 RGB-D 输入的基线对比时提供了预测深度，可能引入附加变量。
