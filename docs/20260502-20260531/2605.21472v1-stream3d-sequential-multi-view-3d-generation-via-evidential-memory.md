## 摘要
基于视图条件的三维生成器，如 SAM 3D、TRELLIS 和 Hunyuan3D，能够从单一视图生成高质量的三维物体重建，但现实世界的视觉观测通常以长单目视频流的形式出现。直接将这些生成器独立应用于每一帧流会导致生成结果中出现严重的时间不一致性。为了解决这一问题，我们提出了 Stream3D，这是首个无需训练的流式机制，可将冻结的视图条件三维生成器转变为具有恒定跨块记忆的流式生成器。Stream3D 通过维护一个紧凑的证据记忆来实现这一目标，该记忆基于所提出的证据评分机制，选择性地缓存信息量最大的历史帧。随着流的推进，记忆动态更新以保留固定数量的信息帧，从而防止记忆占用随序列长度线性增长。这也避免了长序列上的性能退化，并保持底层生成器完全不变，无需重新训练、修改架构或添加辅助损失。在真实和合成流式基准上的评估表明，Stream3D 在光度指标和几何指标上均优于潜在传输基线方法，包括 KV 缓存重用和基于流的特征编辑。更多细节请参见：https://anonymous-submission-20.github.io/streaming3D.github.io/。

## Abstract
View-conditioned 3D generators such as SAM 3D, TRELLIS and Hunyuan3D produce high-quality object reconstructions from a single view, but real-world visual observation often arrives as long monocular streams. Naively applying these generators to each streaming frame independently leads to severe temporal inconsistency in the generated results. To address this problem, we propose Stream3D, the first training-free streaming mechanism that turns a frozen view-conditioned 3D generator into a streaming generator with constant cross-chunk memory. Stream3D achieves this by maintaining a compact evidential memory, which selectively caches the most informative historical frames based on a proposed evidence score mechanism. As the stream progresses, the memory dynamically updates to retain a fixed number of informative frames, preventing the memory footprint from growing linearly with sequence length. This also prevents degradation over long sequences and keeps the underlying generator completely unchanged without retraining, architectural modifications, or auxiliary losses. Evaluated on both realistic and synthetic streaming benchmarks, Stream3D outperforms latent-transport baselines, including KV-cache reuse and flow-based feature editing, across both photometric and geometric metrics. More details can be found at: https://anonymous-submission-20.github.io/streaming3D.github.io/.

## 1. 论文的核心问题与整体含义
- **研究背景**：现代视图条件3D生成器（如SAM 3D、TRELLIS）能从单图重建高质量3D物体，但现实视觉观测往往是长单目视频流。
- **核心问题**：将这类生成器逐帧应用会导致严重的时间不一致性；而直接融合所有视图计算代价高昂，处理固定大小片段又会丢失全局一致性。如何在保持恒定内存、无需重新训练的前提下，让冻结的生成器从流式多视图中生成连续一致的3D内容？
- **整体含义**：将流式3D生成重新定义为**证据选择问题**而非潜在状态传输问题，开辟了从连续视频流中实现长期一致3D生成的新途径，有望推动机器人和视觉监测等实时应用。

## 2. 方法论
- **核心思想**：不跨块传输潜在状态，而是利用生成器自身的交叉注意力图，为每个查询令牌评估历史帧的“证据强度”，并据此动态维护一个容量固定的**自适应证据记忆**。生成时，从记忆中选出信息量最高的K帧进行多视图融合生成。
- **关键技术细节**：
  - **证据评分**：在某一固定层和统一噪声先验下进行单步去噪，提取交叉注意力图 \(A_v\)，对每个查询令牌计算归一化的证据分数 \(M_v[q] \in [0,1]\)，公式涉及注意力的集中度（平均注意力偏离全局平均）与熵的互补：
    \[
    M_v[q] = \left(1 + \left(\sum_i A_v[q,i] - \frac{\sum_{v,i} A_v[q,i]}{N_c}\right)\right) \cdot \left(1 - H(\tilde{A}_v[q])\right)
    \]
    其中 \(\tilde{A}_v[q]\) 是归一化至同一视图的注意力分布，\(H\) 为归一化熵（除以 \(\log P\)）。
  - **自适应证据记忆**：维护两个矩阵 \(M, F \in \mathbb{R}^{Q \times D}\)，分别为证据分数和对应的帧索引，\(D\) 为记忆深度。每收到新帧，对每个令牌将其证据分数与记忆中现存的前 \(D\) 高分进行归并，始终保持分值非递减更新。
  - **条件视图选择**：按每个令牌的记忆“投票”给帧，统计各帧的“拥有令牌数”，选择计数最高的 \(K\) 帧作为**证据视图集**用于生成。
  - **证据多视图生成**：使用所选视图集，通过多扩散融合公式进行前向去噪：
    \[
    \bar{V}_\theta(z_t)[q] = \sum_{v \in V^\star} M_v[q] \, V_\theta(z_t, v)[q]
    \]
    （证据分数在视图中已归一化）生成3D网格。
  - **算法流程**（每个块）：1. 用共享的先验 \(z_0\) 做一次热身前向，提取注意力计算证据分。2. 更新令牌级记忆。3. 统计帧得票选出Top-K视图。4. 进行完整的多视图生成。

## 6. 主要结论与发现
- Stream3D 能够将冻结的单视图3D生成器扩展为流式生成器，在保持恒定内存的同时，产生时间一致、几何准确的3D结果。
- 与单视图生成和直接多视图融合（TRELLIS+M.D.）相比，Stream3D 在所有光度和几何指标上均提升显著。
- 相比于KV缓存、FlowEdit等潜在状态传输方法，Stream3D 通过选择高证据视图而非累积状态，有效避免长期退化。
- 证据评分在较深的网络层更可靠，适当的归一化和记忆深度能进一步改善性能（\(D=3-5\) 较优，\(K=8\) 已足够）。

## 7. 优点
- **免训练，无架构改动**：即插即用，可直接应用于多种冻结的视图条件生成器（如SAM3D）。
- **恒定内存**：记忆占用与流长度无关，支持无限长序列。
- **可解释性**：令牌级证据记忆明确了不同空间区域依赖哪些历史视图，可视化强。
- **长期一致性保障**：证据分值单调非递减，避免质量退化，优于传输潜变量的方案。
- **实验严谨**：双数据集、全面指标、充分消融，对多种流式策略进行对比。

## 8. 不足与局限
- **依赖基础模型性能**：如果基座生成器本身从单视图生成质量差，证据记忆无法挽救缺失的几何或外观。
- **需要外源位姿**：目前依赖Depth Anything 3 估计相机位姿，并非完全端到端。
- **评估限定对象级**：只在物体中心场景测试，未涵盖大规模场景或动态背景。
- **记忆深度固定且启发式**：深度 \(D\) 和选择数 \(K\) 需手动设定，可能对不同场景不是最优。
- **潜在隐患**：未讨论对对抗性视图流或剧烈光照变化的鲁棒性。
