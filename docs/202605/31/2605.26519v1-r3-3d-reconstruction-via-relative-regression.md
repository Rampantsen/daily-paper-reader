## 摘要
近期基于前馈的几何基础模型通过单次前向传播恢复深度和位姿，展现了出色的泛化能力。然而，这些模型通常受限于全局坐标系的假设。这种依赖性成为长序列和流式重建的显著瓶颈，因为它迫使网络维持一个任意的时间原点，并处理随时间无限增长的平移量。我们的解决方案，称为 $R^3$，采用相对回归。我们使用轻量级 MLP 预测置信度加权的相对约束。这些置信度充当统一的锚点：在训练期间加权损失，在推理期间引导位姿聚合。$R^3$ 同时支持全场景离线重建和因果、有界内存的流式处理。我们在离线和流式设置下的评估验证了相对机制的有效性。项目页面：https://kevinxu02.github.io/r3-site

## Abstract
Recent feed-forward geometry foundation models have demonstrated impressive generalization by recovering depth and poses in a single forward pass. However, these models are typically constrained by a global coordinate frame assumption. This dependency becomes a significant bottleneck for long-context and streaming reconstruction, as it forces the network to maintain an arbitrary temporal origin and handle translation magnitudes that grow unbounded over time. Our solution, which we call $R^3$, employs relative regression. We employ a lightweight MLP to predict confidence-weighted relative constraints. These confidences serve as a unified anchor: weighting losses during training and guiding pose aggregation during inference. $R^3$ supports both full-context offline reconstruction and causal, bounded-memory streaming. Our evaluation in both offline and streaming settings validates the effectiveness of our relative mechanism. Project page: https://kevinxu02.github.io/r3-site

# R^3：基于相对回归的三维重建

## 1. 研究动机与背景
- **核心问题**：现有前馈几何基础模型（如 DUSt3R、VGGT、π3、DA3）在单次前向传播中恢复深度和位姿，表现出色，但它们通常依赖一个全局坐标系假设（例如将第一帧作为世界原点）。
- **瓶颈**：这种全局坐标系依赖在长序列视频和流式重建中成为关键瓶颈。
  - 网络必须维持一个任意的时间原点。
  - 平移量随时间无界增长，导致回归目标分布偏移。
  - 在线流式场景下，未来位姿严重偏向早期帧确定的坐标系，难以灵活调整。
- **整体含义**：需要一种新的姿态回归范式，避免直接回归全局位姿，使网络专注于学习局部视图间的相对关系，从而实现更灵活、可扩展的流式三维重建。

## 2. 方法论

### 2.1 核心思想：从全局绝对位姿回归到成对相对位姿回归
- **范式转变**：
  - VGGT 类：固定世界坐标系，只监督从锚点到各帧的有向边。
  - π3：模型自选世界坐标，监督所有无序对，但权重均匀。
  - **R^3**：完全抛弃全局位姿头，预测每两个有序帧对 (i, j) 之间的相对变换（旋转四元数、平移向量），并赋予独立的旋转置信度和平移置信度。
- **姿态图构建**：将所有帧视为节点，预测的成对相对位姿构成一个有向全连接姿态图（图 2c），边属性为相对位姿及两个置信度。全局轨迹由下游的置信度加权聚合步骤拼接而成，不再让网络直接回归全局位姿。

### 2.2 关键技术细节

#### a. 相对位姿预测与置信度设计
- **架构**：在冻结的 DA3 几何骨干网上添加一个共享的轻量级 MLP（配对位姿头）。
- **输入**：两帧经由骨干网提取的相机令牌 $z_i, z_j$。
- **输出**：
  ```math
  (\hat{q}_{i\to j}, \hat{t}_{i\to j}, c^R_{i\to j}, c^T_{i\to j}) = \text{MLP}_{\text{rel}}([z_i; z_j])
  ```
  - $\hat{q}_{i\to j}$：从帧 $i$ 到帧 $j$ 的相对旋转（单位四元数）。
  - $\hat{t}_{i\to j}$：在帧 $i$ 坐标系下帧 $j$ 的平移。
  - $c^R_{i\to j}, c^T_{i\to j} > 0$：分别为旋转和平移的置信度，解耦预测以适应两类误差的不对称性。

#### b. 全局轨迹聚合
- **流式聚合**（有界内存）：
  - 帧 1 固定为世界原点。
  - 对于新帧 $j$，只与**活跃上下文** $C_t$（由帧 1 和关键帧库 $B_t$ 组成）中的参考帧配对。
  - 每个参考帧 $i$ 提出的候选位姿：$q_j^{(i)} = q_i \otimes \hat{q}_{i\to j}$，$t_j^{(i)} = t_i + q_i(\hat{t}_{i\to j})$。
  - 使用 **置信度加权融合**：对旋转和平移分别使用 $c^R$ 和 $c^T$ 得到的 softmax 权重进行加权平均，得到最终位姿。可选用 top-K 个最高平均置信度的参考帧以提升效率，全部参考帧效果更佳。
- **全场景模式**：移除因果掩码，对全部 $N(N-1)$ 个有序对进行预测，先运行一次流式初始化，再进行一次轻量的置信度加权姿态图优化（仅优化位姿，不做 BA）。

#### c. 流式推理与关键帧库管理
- **有界内存**：每帧仅与活跃上下文 $C_t$ 交互，$C_t$ 大小被关键帧库容量 $M_{\max}$ 限制（如 100）。
- **关键帧添加**：新帧 $j$ 的骨干网编码 token 与库中已有 token 的最大余弦相似度低于阈值 $\tau$ 时加入；若长时间无新关键帧（$\Delta_{\max} = 20$）则强制加入以防过时。
- **关键帧剔除**：当库满时，基于效用函数剔除最差帧。效用定义为 $u_j = d_j c_j$，其中 $d_j$ 为 token 级别独特性，$c_j$ 为该帧与库最强的平均置信度。
- **异常帧拒绝**：若新帧与当前上下文的平均置信度低于校准阈值，则丢弃该帧（不污染 KV 缓存和库），并能触发片段重置。

#### d. 训练目标
- **相机损失（置信度感知）**：
  ```math
  L_{\text{rot}}(i,j) = c^R_{i\to j} \ell^{\text{L1}}_{\text{rot}}(i,j) - \alpha \log c^R_{i\to j}
  ```
  ```math
  L_{\text{trans}}(i,j) = c^T_{i\to j} \ell^{\text{L1}}_{\text{trans}}(i,j) - \alpha \log c^T_{i\to j}
  ```
  - 损失项 $\alpha \log c$ 防止置信度坍缩为零。$\alpha=0.2$。
  - 在因果训练中，仅对过去到当前的定向对计算损失。
- **深度损失**：在中值归一化空间内，对每个像素使用学习到的置信度进行加权 L1 损失，监督目标来自真实深度（合成数据）或冻结的 DA3 教师模型输出（真实数据）。
- **联合训练**：总损失 $L = L_{\text{cam}} + L_{\text{depth}}$，骨干大部分冻结，仅更新全局注意力层和新增头。

## 6. 主要结论与发现
- **位姿精度**：在 372M 参数下，流式 R^3 在三个基准上的流式位姿估计整体最优，全场景模式也与离线大型模型（VGGT/DA3）具有竞争力。
- **稠密重建**：在 7-Scenes 和 NRGBD 上达到或接近最佳流式方法，全场景模式进一步提升指标。
- **长序列扩展**：R^3 是少数在千帧级别保持稳定且不超内存的方法之一，无漂移增长明显优于对比方法（如 ATE 曲线），在 DL3DV 户外长序列上以巨大优势领先。
- **鲁棒性**：利用置信度门可在单次在线传播中高效识别并剔除干扰帧，胜过需要多步离线处理的 RobustVGGT。
- **置信度的双重作用**：训练时作为损失权重，推理时作为聚合权重和关键帧库管理信号，验证了置信度作为统一基元的有效性。

## 7. 优点
- **范式创新**：将前馈重建从全局绝对回归转向成对相对回归，解决了长序列中坐标系选择难题。
- **统一置信度基元**：旋转和平移独立置信度设计，贯穿训练、聚合、关键帧选择、异常剔除，节省设计并增加一致性。
- **单模型多模式**：同一因果 checkpoint 可同时支持有界内存流式和全场景离线推理，只需简单的测试时注意力掩码切换。
- **轻量与高效**：参数量仅 372M，训练资源需求低（6×48GB），推理可在 20+ FPS 运行且能处理数千帧。
- **强长序列鲁棒性**：在多种流式方法随长度增加而漂移或内存溢出时，R^3 表现出稳定的位姿和重建质量。
- **全面的实验验证**：覆盖多种场景类型和对比基线，消融和鲁棒性分析增强了结论可信度。

## 8. 不足与局限
- **部分手工阈值依赖**：关键帧库操作仍依赖少量手工设置阈值（$\tau, \Delta_{\max}, M_{\max}$），虽然在当前基准上泛化良好，但迁移到非常不同的领域时可能需要重新调整。
- **弱遮挡/远距离视角残留不一致**：由于纯因果流式无法回溯过去 token，在遮挡和远距离视角下几何重建可能残留不一致。
- **长流式漂移风险**：尽管优于对比方法，在极长序列上无重置时仍可能出现漂移，论文中已展示需要周期性分段重置来保持干净轨迹（图 9）。
- **训练数据与模型规模未探索**：未尝试更大骨干网络或更多数据，未来缩放空间尚不明确。
- **仅姿态图优化**：全场景后处理为轻量姿态图优化，无捆绑调整或深度优化，可能限制绝对精度。
- **对比方法覆盖**：Concurrent work 如 LingBot-Map 因训练资源规模巨大未纳入公平比较。
