## 摘要
本文介绍了 S23DR 2026 挑战赛的获胜方案，该挑战赛旨在从稀疏运动恢复结构（SfM）、拟合深度和语义分割中重建结构化三维线框。该方法将顶点视为条件集合，并使用基于 Perceiver 风格场景标记条件化的流匹配 DiT 模型对 64 个顶点标记进行去噪。全局推理阶段预测粗略结构，基于外壳裁剪的第二阶段对其进行细化，最后通过小型多样本一致性步骤使随机采样器保持良好行为。最终系统在私有排行榜上排名第一，获得了 HSS = 0.654 的成绩。

## Abstract
This text presents the winning solution to the S23DR 2026 challenge for structured 3D wireframe reconstruction from sparse SfM, fitted depth, and semantic segmentations. The method treats vertices as a conditional set and denoises 64 vertex tokens with a flow-matching DiT conditioned on Perceiver-style scene tokens. A global pass predicts the coarse structure, a hull-cropped second pass refines it, and a small multi-sample consensus step keeps the stochastic sampler well behaved. The final system ranked first on the private leaderboard, achievingHSS = 0.654.

### 2. 方法论
- **核心思想**：将顶点预测视为一个**条件集合生成问题**，而不是传统的 2D‑to‑3D 提升方法。用一个固定大小的顶点 slot 集合表示待重建的线框，通过流匹配（flow matching）去噪过程逐步将这些 slot 从随机位置“移动”到真实的 3D 顶点位置。
- **关键技术细节**：
    - **场景编码器（Perceiver 风格）**：对每个点（COLMAP 点、深度点、相机中心）进行多特征编码（xyz、Fourier 特征、来源类型、Gestalt/ADE20k 语义类、置信度等），经两层全分辨率自注意力后，由 1024 个可学习 token 通过交叉注意力汇聚成固定数量的场景上下文 token。
    - **流匹配 DiT 去噪器**：在 64 个顶点 token 上操作，每个 token 含有当前 3D 位置。DiT block 以时间嵌入和场景 token 作为条件，输出三维速度、有效性 logit 和两两之间的边 logit。训练时使用线性插值定义流：`x_t = (1-t)x_0 + t x_1` 且目标速度 `v* = x_1 - x_0`。顶点通过匈牙利匹配与真值对齐，损失函数结合 SmoothL1 速度损失、端点损失、focal 有效性损失和 BCE 边损失。
    - **两阶段流程**：
        - **阶段一**（全局生成）：从整个场景随机采样的 8192 个点出发，预测粗线框顶点。
        - **阶段二**（外壳裁剪细化）：基于阶段一顶点构造凸包并膨胀，裁剪出仅包含屋顶区域的更密集点云（16384 点），以第一阶段顶点初始化 slot，进行第二次精细化去噪。
    - **集成推理**：运行 16 条随机轨迹，取依据匈牙利距离和边一致性的 medoid 预测，再用跨样本一致性聚合顶点位置，降低方差。
- **公式与算法概要**：线性流 `x_t = (1-t)x_t + t x_1`，损失 `L = L_flow + 0.5 L_end + 0.2 L_valid + 0.2 L_edge`；推理用 50 步 Euler 积分求解常微分方程。

### 6. 主要结论与发现
- 方案在 S23DR 2026 私有测试集上取得了 **HSS 0.654**，赢得冠军，同时在顶点 F1（0.791）上也位于所有提交方案之首。
- 将线框重建建模为流匹配下条件集合生成，优于传统的 2D-3D 提升方法。
- 全局粗预测加局部裁剪细化的两阶段设计有效提升了在强噪声环境下的几何精度与拓扑完整性。

### 7. 优点
- **新颖的任务建模**：将无序、不定数量的顶点重建问题优雅地表达为固定容量 slot 的条件生成，借助匈牙利匹配实现排列不变监督，避免了回归固定数量点带来的对齐困难。
- **高效去噪与条件机制**：流匹配 DiT 在连续时间中学习速度场，交叉注意力引入 Perceiver 场景 token，能在稀疏观测下有效定位顶点。
- **两阶段细化**：通过外壳裁剪滤除远处无关点，阶段二可以集中容量进行局部微调，兼顾全局结构与局部精度。
- **工程细节扎实**：丰富的点特征编码、基于优先级的点采样策略、合理的损失权重设计、轻量的集成后处理，使系统稳健且推理速度可接受。
- **竞赛验证充分**：在难以获取额外测试数据的竞赛场景下，通过与官方基线的对比和消融实验证实了方法有效性。

### 8. 不足与局限
- **场景泛化能力未充分验证**：仅针对 HoHo 数据集中的住宅屋顶，未在工业建筑、非规则屋顶或更开放场景下测试，迁移性存疑。
- **对输入质量的依赖**：模型假设 SfM 点云、深度和语义分割存在，但实际中深度大量缺失（验证集缺失率 69%），对缺失情况的稳健性未做专门分析；当成像质量更差或 SfM 大幅失败时性能可能骤降。
